O que é taxa de contraste?

A taxa de contraste mede a diferença de luminância entre as partes mais claras e mais escuras de uma imagem ou interface. Uma relação de contraste de 21:1 representa o contraste máximo (branco puro sobre preto puro), enquanto 1:1 representa nenhum contraste (cores idênticas). As Diretrizes de Acessibilidade para Conteúdo da Web (WCAG) exigem taxas de contraste mínimas para texto legível.

Os web designers usam taxas de contraste para garantir que o texto seja legível para usuários com deficiência visual. Texto preto (#000000) sobre fundo branco (#FFFFFF) atinge contraste de 21:1 – excedendo em muito o requisito WCAG AA de 4,5:1 para texto normal. O texto cinza claro (#999999) em branco atinge apenas 2,8:1 – falhando nos padrões de acessibilidade e sendo difícil de ler para muitos usuários.

Os desenvolvedores de UI verificam as taxas de contraste para elementos interativos, como botões e links. Um botão azul (#4A90D9) com texto branco atinge contraste de 4,6:1 – passando WCAG AA. O mesmo azul com texto cinza claro (#CCCCCC) cai para 1,8:1 – falhando e potencialmente inutilizável para usuários com baixa visão ou daltonismo.

Fórmulas de taxa de contraste com cálculos trabalhados

WCAG 2.1 define a taxa de contraste usando luminância relativa:

CR = (L₁ + 0,05) / (L₂ + 0,05)

Onde L₁ é a luminância relativa da cor mais clara (0 a 1) e L₂ é a luminância relativa da cor mais escura (0 a 1).

A luminância relativa é calculada a partir dos valores sRGB:

L = 0,2126 × R + 0,7152 × G + 0,0722 × B

Onde R, G e B são os valores de cores linearizados (corrigidos por gama).

Exemplo resolvido:Calcule o contraste entre #3366CC (azul) e #FFFFFF (branco).

Primeiro, converta hexadecimal em sRGB (escala 0-255): #3366CC = rgb(51, 102, 204), #FFFFFF = rgb(255, 255, 255).

Normalize para a escala 0-1: R = 51/255 = 0,2, G = 102/255 = 0,4, B = 204/255 = 0,8 para azul. Branco é R = G = B = 1.

Linearize cada canal (correção gama inversa):

Se valor ≤ 0,03928: linear = valor / 12,92

Se valor > 0,03928: linear = ((valor + 0,055) / 1,055)^2,4

Para azul: R = ((0,2 + 0,055) / 1,055) ^ 2,4 = 0,0337, G = ((0,4 + 0,055) / 1,055) ^ 2,4 = 0,1369, B = ((0,8 + 0,055) / 1,055) ^ 2,4 = 0,6072

Para branco: R = G = B = ((1 + 0,055) / 1,055)^2,4 = 1,0

Calcule a luminância relativa:

L_azul = 0,2126 × 0,0337 + 0,7152 × 0,1369 + 0,0722 × 0,6072 = 0,0072 + 0,0979 + 0,0438 = 0,1489

L_branco = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 1 = 1,0

Calcular taxa de contraste:

CR = (1,0 + 0,05) / (0,1489 + 0,05) = 1,05 / 0,1989 = 5,28:1

Resultado: Azul #3366CC em branco #FFFFFF atinge contraste de 5,28:1 — passa WCAG AA (4,5:1) e AAA (3:1) para texto normal.

Como calcular a taxa de contraste: 6 etapas

  1. Converter cores em RGB:A cor do primeiro plano é #7B2D8E (roxo) e do fundo é #F5F5F5 (cinza claro). Converter: #7B2D8E = rgb(123, 45, 142), #F5F5F5 = rgb(245, 245, 245).
  2. Normalize para escala 0-1:Roxo: R = 123/255 = 0,482, G = 45/255 = 0,176, B = 142/255 = 0,557. Cinza: R = G = B = 245/255 = 0,961.
  3. Aplicar correção gama:Para roxo: R = ((0,482 + 0,055) / 1,055) ^ 2,4 = 0,203, G = ((0,176 + 0,055) / 1,055) ^ 2,4 = 0,026, B = ((0,557 + 0,055) / 1,055) ^ 2,4 = 0,272. Para cinza: R = G = B = ((0,961 + 0,055) / 1,055)^2,4 = 0,919.
  4. Calcule a luminância relativa:L_roxo = 0,2126 × 0,203 + 0,7152 × 0,026 + 0,0722 × 0,272 = 0,043 + 0,019 + 0,020 = 0,082. L_cinza = 0,2126 × 0,919 + 0,7152 × 0,919 + 0,0722 × 0,919 = 0,919.
  5. Identifique o mais claro e o mais escuro:L_cinza (0,919) > L_purple (0,082), então L₁ = 0,919 e L₂ = 0,082.
  6. Aplicar fórmula de contraste:CR = (0,919 + 0,05) / (0,082 + 0,05) = 0,969 / 0,132 = 7,34:1. Este roxo em cinza claro atinge 7,34:1 – passa WCAG AAA (7:1) para texto normal.

5 exemplos de taxa de contraste

Exemplo 1 — Preto sobre branco (contraste máximo):#000000 em #FFFFFF. L_black = 0 (todos os canais linearizam para 0). L_branco = 1. CR = (1 + 0,05) / (0 + 0,05) = 1,05 / 0,05 = 21:1. Este é o contraste máximo possível no espaço de cores sRGB – o padrão ouro para legibilidade.

Exemplo 2 — Cinza sobre cinza (falta de contraste):#999999 em #CCCCCC. L_light = 0,2126 × 0,502 + 0,7152 × 0,502 + 0,0722 × 0,502 = 0,502. L_escuro = 0,2126 × 0,282 + 0,7152 × 0,282 + 0,0722 × 0,282 = 0,282. CR = (0,502 + 0,05) / (0,282 + 0,05) = 0,552 / 0,332 = 1,66:1. Isso falha em todos os níveis WCAG - comum em interfaces mal projetadas, onde o texto "sutil" cinza sobre cinza se torna ilegível.

Exemplo 3 — Verde sobre branco (limite AA):#00AA00 (verde brilhante) em #FFFFFF. L_verde = 0,2126 × 0 + 0,7152 × 0,331 + 0,0722 × 0 = 0,237. L_branco = 1. CR = (1 + 0,05) / (0,237 + 0,05) = 1,05 / 0,287 = 3,66:1. Isso falha nas WCAG AA (4.5:1) para texto normal, mas é aprovado para texto grande (18pt ou 14pt em negrito, exigindo 3:1). O texto verde é particularmente problemático para usuários daltônicos vermelho-verde.

Exemplo 4 — Amarelo sobre preto (alto contraste):#FFFF00 (amarelo puro) em #000000. L_amarelo = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 0 = 0,928. L_preto = 0. CR = (0,928 + 0,05) / (0 + 0,05) = 0,978 / 0,05 = 19,56:1. Amarelo sobre preto atinge contraste quase máximo – por que os sinais de alerta e as etiquetas de perigo usam essa combinação para obter visibilidade máxima.

Exemplo 5 — Link azul em branco (padrão web comum):#0000EE (link padrão azul) em #FFFFFF. L_azul = 0,2126 × 0 + 0,7152 × 0,033 + 0,0722 × 0,549 = 0,024 + 0,040 = 0,064. L_branco = 1. CR = (1 + 0,05) / (0,064 + 0,05) = 1,05 / 0,114 = 9,21:1. O link azul tradicional passa facilmente em todos os níveis WCAG – um dos motivos pelos quais se tornou o padrão da web na década de 1990 e permanece acessível até hoje.

4 erros comuns na taxa de contraste

  • Usando fórmulas simples de luminância:O cálculo da luminância como (R + G + B) / 3 ignora a sensibilidade diferente da visão humana para cada cor. O verde contribui com 71,52% do brilho percebido, o vermelho com 21,26% e o azul com apenas 7,22%. Uma cor como #00FF00 (verde puro) parece muito mais brilhante que #0000FF (azul puro), apesar de somas RGB iguais.
  • Ignorando a correção gama:Usar valores sRGB diretamente sem linearização produz luminância incorreta. O espaço de cores sRGB usa uma curva gama para corresponder à percepção humana. #808080 (cinza médio) tem luminância linear de 0,216, e não de 0,5 — uma diferença de 57% que afeta significativamente os cálculos de contraste.
  • Testando apenas tamanhos de texto normais:WCAG tem requisitos diferentes para texto normal (4,5:1 AA, 7:1 AAA) versus texto grande (3:1 AA, 4,5:1 AAA). O texto grande tem 18 pontos ou 14 pontos em negrito. Um contraste de 3,5:1 falha em AA para corpo de texto de 12 pontos, mas é aprovado para títulos de 20 pontos.
  • Ignorando elementos não textuais:WCAG também requer contraste 3:1 para componentes de UI (botões, bordas de entrada) e objetos gráficos (ícones, gráficos). Uma borda de botão cinza (#CCCCCC) sobre fundo branco (#FFFFFF) atinge apenas 1,3:1 – falhando no requisito de 3:1 e tornando o botão invisível para muitos usuários.

5 dicas para conformidade com a taxa de contraste

  • Comece com padrões de alto contraste:Use #000000 em #FFFFFF (21:1) ou #1A1A1A em #FAFAFA (18:1) como base. Estes excedem todos os requisitos e fornecem espaço para variações de design. Reduza o contraste intencionalmente apenas para elementos de design específicos, nunca para o corpo do texto.
  • Use DevTools do navegador:Os DevTools do Chrome e do Firefox mostram taxas de contraste no seletor de cores. Clique em qualquer cor no painel Estilos e o seletor exibirá a taxa de contraste em relação ao plano de fundo. Ele também mostra aprovação/reprovação para WCAG AA e AAA rapidamente.
  • Teste com simuladores de daltonismo:Ferramentas como Stark, Color Oracle ou configurações de renderização do Chrome simulam vários tipos de deficiência de visão de cores. Um contraste vermelho-verde que passa matematicamente (4,5:1) ainda pode ser ilegível para usuários deuteranópicos se a tonalidade for o único fator distintivo.
  • Verifique os estados de foco e foco:Um botão com contraste de 5:1 em seu estado padrão pode cair para 2:1 ao passar o mouse se o fundo ficar mais claro. Teste todos os estados interativos: padrão, foco, foco, ativo e desativado. Cada estado deve atender aos requisitos mínimos de contraste.
  • Use ferramentas de auditoria automatizadas:Lighthouse (integrado ao Chrome), ax DevTools ou WAVE podem verificar páginas inteiras em busca de violações de contraste. Essas ferramentas detectam problemas que falham em testes manuais, como conteúdo gerado dinamicamente ou variáveis ​​CSS que alteram o contraste com base no tema.

Primeiro, componha a cor transparente sobre o fundo usando a mistura alfa. Para rgba(0, 0, 0, 0,5) sobre #FFFFFF: RGB resultante = (0 × 0,5 + 255 × 0,5) = 127,5 → #808080. Em seguida, calcule o contraste entre a cor composta (#808080) e o fundo final. A transparência em si não afeta o contraste – apenas a cor visível final é importante.

A constante 0,05 leva em conta o brilho da tela e os reflexos da luz ambiente. Em condições de visualização do mundo real, mesmo o “preto puro” em uma tela reflete alguma luz, e o “branco puro” não parece infinitamente brilhante. O deslocamento evita taxas de contraste infinitas (divisão por zero) e modela ambientes de visualização típicos com luz ambiente moderada.
