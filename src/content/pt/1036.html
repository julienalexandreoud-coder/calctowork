O que é um rolo de dados?

Um rolo de dados simula o lançamento de dados físicos usando geração de números aleatórios, produzindo resultados imprevisíveis para jogos de mesa, experimentos de probabilidade e tomada de decisões. Desde campanhas de Dungeons & Dragons que exigem jogadas de dano de 4d6+3 até aulas de estatística que demonstram o teorema central do limite, os lançamentos de dados digitais fornecem resultados aleatórios rápidos, justos e verificáveis, sem a necessidade de dados físicos.

Você também pode encontrar oRandom Number Generator and Coin Flip useful.

A matemática por trás do lançamento de dados revela por que os rolos digitais são importantes. Um dado justo de seis lados (d6) tem probabilidade igual de 1/6 (16,67%) para cada face. Mas os dados fabricados apresentam preconceito – os dados de cassino custam mais de US$ 20 porque são balanceados com precisão para uma tolerância de 0,0005 polegadas. Dados baratos de jogos de tabuleiro podem apresentar tendência de 5% em relação a certas faces devido à distribuição desigual do peso dos pips. Os rolos digitais que usam RNG criptograficamente seguro eliminam totalmente o preconceito físico, garantindo uma probabilidade real de 1/6 para cada lançamento.

A notação dos dados padroniza como os lançamentos são expressos: "2d6" significa lançar dois dados de seis lados e somar; "1d20+5" significa lançar um dado de vinte lados e adicionar 5; "4d6kh3" significa lançar quatro dados de seis lados e manter o 3 mais alto. Essa notação, originada de RPGs de mesa na década de 1970, agora aparece em documentos de design de jogos, calculadoras de probabilidade e APIs de jogos em todo o mundo. Compreender a notação dos dados permite uma comunicação precisa sobre resultados aleatórios nas comunidades de jogos.

Explicação da fórmula e probabilidade do lançamento de dados

Fórmula básica de lançamento de dados:

Resultado = Σ(random(1, s) para cada dado) + modificador

Where:

  • s = Número de lados por dado (4, 6, 8, 10, 12, 20, 100, etc.)
  • n = Número de dados lançados
  • modificador = Constante adicionada ou subtraída (pode ser 0)

Análise da notação de dados:

  • XdY = Jogue X dados com Y lados cada, some os resultados
  • XdY+Z = Jogue X dados com Y lados, adicione Z ao total
  • XdYkhZ = Jogue X dados com Y lados, mantenha os dados Z mais altos
  • XdYklZ = Jogue X dados com lados Y, mantenha os dados Z mais baixos
  • XdY! = Jogue X dados com Y lados, exploda no máximo (role novamente e adicione)

Distribuição de probabilidade para XdY:

Para um único dado (1dY): Distribuição uniforme, P(qualquer valor) = 1/Y

Para dados múltiplos (XdY onde X > 1): Distribuição triangular ou em forma de sino

Valor esperado (média) para XdY:

E[XdY] = X × (Y + 1) / 2

Variação para XdY:

Var(XdY) = X × (Y² - 1) / 12

Exemplo resolvido 1 - jogada de ataque padrão de D&D (1d20 + 5):

Passo 1: Jogue um dado de 20 faces: random(1, 20) = 17

Etapa 2: Adicionar modificador: 17 + 5 = 22

Resultado: 22(atinge AC 22 ou menos)

Etapa 3: Calcule a probabilidade de acertar AC 18:

Necessário: 1d20 + 5 ≥ 18, então 1d20 ≥ 13

Sucesso nas jogadas 13-20: 8 resultados em 20

P(acerto) = 8/20 = 0,40 = 40%

Exemplo resolvido 2 - Estatísticas do personagem (4d6kh3):

Passo 1: Jogue quatro dados de 6 faces: [4, 6, 3, 5]

Etapa 2: Classificar em ordem decrescente: [6, 5, 4, 3]

Etapa 3: Mantenha o 3 mais alto: 6 + 5 + 4 = 15

Resultado: 15(estatística acima da média, a média é 12,24)

Etapa 4: Calcule o valor esperado para 4d6kh3:

E[4d6kh3] = 4 × 3,5 - E[menor de 4d6] = 14 - 1,76 = 12,24

Exemplo resolvido 3 — Dano de bola de fogo (8d6):

Etapa 1: jogue oito dados de 6 lados: [3, 6, 2, 5, 4, 6, 1, 5]

Etapa 2: Soma todos os valores: 3 + 6 + 2 + 5 + 4 + 6 + 1 + 5 = 32

Resultado: 32 de dano

Etapa 3: Calcular o dano esperado:

E[8d6] = 8 × (6 + 1) / 2 = 8 × 3,5 = 28

Etapa 4: Calcular o desvio padrão:

Var(8d6) = 8 × (36 - 1) / 12 = 8 × 2,917 = 23,33

SD = √23,33 = 4,83

Alcance típico (±1 SD): 23 a 33 de dano (68% das jogadas)

6 etapas para lançar dados

  1. Identifique a notação dos dados ou os requisitos de lançamento:Analise a expressão do lançamento: "3d8 + 2" significa 3 dados, 8 lados cada, modificador +2. "2d20kh1" significa 2 dados de vinte faces, mantenha o maior 1 (vantagem em D&D 5e). “5d10!” significa 5 dados de dez lados com explosão (rolar novamente os valores máximos). Anote cada componente: número de dados, lados por dado, modificador e quaisquer regras especiais (manter o mais alto/mais baixo, explodir, rolar novamente).
  2. Gere valores aleatórios para cada dado:Para cada dado, gere um número inteiro aleatório de 1 ao número de lados. Use um RNG justo – crypto.getRandomValues() para rolos online ou RNG de hardware para aplicativos de cassino. Para 3d8: gere três valores, por exemplo, [7, 3, 8]. Cada valor tem probabilidade igual a 1/8 (12,5%). Registre os resultados individuais das matrizes para transparência e verificação.
  3. Aplicar regras especiais, se aplicável:Para "manter o mais alto" (kh): classifique os dados em ordem decrescente, some apenas os valores Z superiores. Para "manter o mais baixo" (kl): classificação crescente, soma dos valores Z inferiores. Para "explodir" (!): se algum dado mostrar o valor máximo, jogue um dado adicional e adicione-o (repita se o novo dado também atingir o máximo). Para "reroll 1s": substitua quaisquer 1s por novos lançamentos. Aplique estas regras antes de somar.
  4. Some os dados relevantes:Some os valores dos dados que estão sendo contados (todos os dados para lançamentos padrão, dados mantidos para lançamentos kh/kl, incluindo dados de explosão). Para 3d8 rolando [7, 3, 8]: soma = 7 + 3 + 8 = 18. Para 4d6kh3 rolando [4, 6, 3, 5], mantendo [6, 5, 4]: soma = 15. Verifique novamente a adição - erros aritméticos simples frustram mais os jogadores do que azar.
  5. Adicione quaisquer modificadores:Aplique o modificador constante da notação. Para 3d8+2 com soma 18: 18 + 2 = 20. Os modificadores podem ser negativos (1d20-3 para jogadas de desvantagem) ou zero (lançamentos de dano puro). Alguns sistemas aplicam modificadores de maneira diferente – GURPS adiciona modificadores antes de comparar com o alvo, enquanto D&D aumenta a rolagem. Siga as regras específicas do jogo.
  6. Exiba e registre o resultado:Mostre o total final junto com os valores individuais dos dados para maior transparência. "3d8+2 = [7, 3, 8] + 2 = 20" permite aos jogadores verificar se o lançamento não foi manipulado. Para registros de jogos ou registros de campanha, inclua carimbo de data e hora, nome do jogador e propósito do teste ("Rolada de ataque Goblin vs. AC 16"). Isso cria uma trilha auditável para resultados contestados.

5 exemplos práticos de lançamento de dados

Exemplo 1 — Jogada de Ataque D&D 5e com Vantagem (2d20kh1+7):

  • Role: 2 dados de vinte faces, mantenha o mais alto, adicione 7
  • Lançamentos individuais: [14, 19]
  • Manter o mais alto: 19
  • Adicionar modificador: 19 + 7 = 26
  • Resultado: 26(atinge a maioria dos monstros em D&D)
  • Análise de probabilidade: A vantagem aumenta a chance de acerto de 60% para 84% vs. AC 18

Exemplo 2 — Rolagem de Dano do Pathfinder (2d6+4 para espada longa):

  • Jogue: 2 dados de seis faces, adicione 4
  • Rolos individuais: [3, 5]
  • Soma: 3 + 5 = 8
  • Adicionar modificador: 8 + 4 = 12
  • Resultado: 12 danos cortantes
  • Dano esperado: E[2d6+4] = 2 × 3,5 + 4 = 11 (este teste está acima da média)

Exemplo 3 — Dados Explosivos de Mundos Selvagens (1d10!):

  • Rolamento: 1 dado de dez lados, explode em 10
  • Primeiro lançamento: 10 (máx. Role novamente e adicione)
  • Rolagem de explosão: 7
  • Total: 10 + 7 = 17
  • Resultado: 17(excelente rolagem, o máximo sem explosão é 10)
  • Probabilidade de pelo menos uma explosão: 10% por rolo, teto infinito

Exemplo 4 — Teste de Perícia GURPS (3d6 ≤ perícia):

  • Lançamento: 3 dados de seis faces, a soma deve ser ≤ nível de habilidade
  • Rolos individuais: [2, 4, 3]
  • Soma: 2 + 4 + 3 = 9
  • Result: 9(sucesso se habilidade ≥ 9, sucesso crítico se habilidade ≥ 16)
  • Distribuição de probabilidade: 3d6 produz uma curva em sino centrada em 10,5
  • P(rolagem ≤ 10) = 50%, P(rolagem ≤ 14) = 90,7%, P(rolagem = 3 ou 18) = 0,46% cada

Exemplo 5 — Warhammer 6+ Armor Save (5d6, contagem de sucessos ≥ 5):

  • Jogue: 5 dados de seis lados, conte os dados mostrando 5 ou 6
  • Rolos individuais: [2, 5, 6, 3, 6]
  • Sucessos (≥5): dado 2 (5), dado 3 (6), dado 5 (6) = 3 sucessos
  • Resultado: 3 defesas feitas, 2 ferimentos sofridos
  • Salvamentos esperados: 5 × (2/6) = 1,67 salvamentos (este lançamento superou a expectativa)

4 erros comuns no lançamento de dados

  • Usando geradores de números aleatórios tendenciosos:Alguns rolos de dados online usam Math.random() ou PRNGs não criptográficos semelhantes que podem produzir padrões em milhares de lançamentos. Embora seja aceitável para jogos casuais, isso é importante para jogos de azar ou jogos competitivos onde a imparcialidade é examinada. Use rolos que declarem explicitamente que usam RNG criptograficamente seguro ou que sejam auditados por terceiros. Os dados físicos têm viés de fabricação; os dados digitais têm viés algorítmico se mal implementados.
  • Aplicação incorreta das regras de vantagem/desvantagem:Em D&D 5e, vantagem significa rolar 2d20 e pegar o maior (2d20kh1), NÃO rolar 1d20 + 5 ou rolar duas vezes e adicionar ambos. Desvantagem significa pegar o menor (2d20kl1). Múltiplas fontes de vantagem não se acumulam – você ainda rola apenas 2d20. Um erro comum é rolar 3d20 quando você tem duas fontes de “vantagem”. A regra é binária: qualquer número de vantagens = role 2d20kh1, qualquer número de desvantagens = role 2d20kl1.
  • Esquecendo de redefinir as cadeias de explosão:Dados explosivos (rolar novamente no máximo) podem encadear indefinidamente. Um jogador rola 10 em 1d10!, depois rola outro 10, depois outro 10, depois um 3. Total = 10 + 10 + 10 + 3 = 33, não 13. Acompanhe cada explosão separadamente. Alguns sistemas limitam as explosões a uma vez por dado ou limitam o valor total. Sempre esclareça as regras de explosão antes de rolar para evitar disputas no meio do jogo sobre se aquela jogada de mais de 50 danos foi legal.
  • Notação de dados confusa em sistemas de jogo:"4d6" significa coisas diferentes em contextos diferentes. Na criação de personagens de D&D, geralmente significa "4d6kh3" (role 4, mantenha o maior 3). Nas jogadas de dano, significa somar todos os 4 dados. Em Shadowrun, "6d6" significa contar quantos dados mostram 5 ou 6 (sucessos), não soma. Sempre especifique a notação completa, incluindo regras especiais. Em caso de dúvida, escreva o procedimento completo em vez de assumir um entendimento compartilhado da notação abreviada.

5 dicas profissionais para lançar dados

  • Entenda as curvas de probabilidade para equilíbrio do jogo:Dados únicos (1d20) produzem probabilidade fixa – todos os resultados são igualmente prováveis. Múltiplos dados (3d6, 4d6kh3) produzem curvas em forma de sino – resultados médios são comuns, extremos raros. Isso afeta o design do jogo: o 1d20 de D&D significa que um modificador +1 altera a taxa de sucesso em exatamente 5%. 3d6 do GURPS significa +1 altera a taxa de sucesso em 12,5% na média, mas apenas 2,8% nos extremos. Escolha a mecânica que corresponda ao nível de aleatoriedade desejado.
  • Use rolos de dados com histórico e estatísticas de lançamento:Bons rolos digitais rastreiam todos os rolos, mostrando a distribuição ao longo do tempo. Depois de 100 jogadas de ataque, você realmente conseguiu cinco 1s naturais? A história prova (ou refuta) que “os dados me odeiam”. Alguns vídeos mostram estatísticas em tempo real: média de lançamento, desvio padrão, frequência de cada resultado. Isso ajuda a identificar preconceitos reais versus padrões percebidos. Os humanos se lembram mais dos fracassos dramáticos (nat 1) do que dos sucessos mundanos, criando falsas impressões de injustiça.
  • Expressões de dano complexo precedente:Para feitiços com dano variável (bola de fogo 8d6), faça um pré-rolamento durante o turno do jogador anterior para acelerar o jogo. Muitas mesas virtuais (Roll20, Foundry) permitem criar botões macro para expressões roladas com frequência. O botão "Bola de fogo" executa 8d6 instantaneamente. Isso mantém o fluxo do combate e reduz a ocorrência de erros matemáticos. Para jogo físico, jogue os dados de dano junto com os dados de ataque se tiver certeza de que acertará.
  • Implemente conjuntos de dados com contagem de sucesso para lançamentos grandes:Rolar 20d6 e somar é tedioso. Para grandes conjuntos de dados, use sistemas de contagem de sucessos: cada dado mostrando 4+ = 1 sucesso. Rolar 20d6 para "contar 4+" é mais rápido do que somar. Sucessos esperados = 20 × 0,5 = 10. Muitos sistemas (World of Darkness, Shadowrun) usam essa abordagem especificamente para escalabilidade. Os rolos digitais lidam com qualquer tamanho de pool instantaneamente, mas entender a matemática ajuda a definir CDs razoáveis.
  • Defina números DC/alvo usando matemática de probabilidade:Em vez de adivinhar a dificuldade, calcule o lançamento necessário e a probabilidade correspondente. Para sistema 3d6 com média de 10,5: CD 11 = 50% de sucesso, CD 14 = 25% de sucesso, CD 17 = 5% de sucesso. Para 1d20+5 vs. CA: CA 15 = 55% de acerto, CA 20 = 30% de acerto, CA 25 = 5% de acerto. Projete encontros com taxas de sucesso pretendidas, não com números arbitrários. Uma verificação "difícil" deve ser bem-sucedida em aproximadamente 25-40% das vezes; uma verificação "quase impossível" de aproximadamente 5 a 10%.

4 perguntas frequentes sobre lançamento de dados

Rolos digitais bem projetados usando RNG criptograficamente seguro produzem resultados distribuídos uniformemente, indistinguíveis da verdadeira aleatoriedade. Cada face tem probabilidade exata de 1/n, verificada por meio de testes estatísticos. Os dados físicos, mesmo os dados de cassino de precisão, apresentam um leve viés devido a imperfeições de fabricação. No entanto, a confiança é importante: use vídeos de fontes confiáveis ​​que publicam sua metodologia RNG e sejam submetidos a auditorias de terceiros. Os rolos de código fechado poderiam, teoricamente, manipular os resultados.

Valores esperados: 1d4 = 2,5, 1d6 = 3,5, 1d8 = 4,5, 1d10 = 5,5, 1d12 = 6,5, 1d20 = 10,5. Para dados múltiplos: 2d6 = 7, 3d6 = 10,5, 4d6 = 14, 4d6kh3 = 12,24, 2d10 = 11, 2d20 = 21. Adicione modificadores após calcular a expectativa base. Estas são médias de longo prazo – os resultados individuais variam amplamente. O desvio padrão para 1d20 é 5,77, o que significa que ~68% dos lançamentos ficam dentro de ±6 da média (4-17 para 1d20).

A maioria das linguagens possui funções aleatórias integradas. Python: random.randint(1, 6) para 1d6. JavaScript: Math.floor(Math.random() * 6) + 1. Para dados múltiplos: sum([random.randint(1, 6) for _ in range(3)]) para 3d6. Para rolos criptograficamente seguros: módulo de segredos do Python (secrets.randbelow(6) + 1) ou crypto.getRandomValues() do JavaScript. Sempre propague PRNGs adequadamente – use o tempo do sistema para jogos, sementes fixas para testes reproduzíveis.

Legalmente, isso depende da jurisdição. Muitas regiões exigem que os RNGs de jogos de azar sejam certificados por laboratórios de testes aprovados (GLI, BMM Testlabs) e usem RNGs de hardware com monitoramento de entropia. Usar um rolo online não certificado para jogos de azar com dinheiro real pode violar os termos de serviço e as leis locais. Para jogos com dinheiro fictício ou jogos casuais entre amigos, qualquer apostador justo funciona. Para aplicações de jogos de azar sérias, invista em hardware RNG certificado e passe pela aprovação regulatória.
