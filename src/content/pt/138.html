O que é a calculadora mediana?

A Calculadora de Mediana encontra o valor médio em um conjunto de dados classificado, fornecendo uma medida de tendência central que resiste à distorção de valores extremos. Ao contrário da média aritmética, que pode ser distorcida por valores invulgarmente altos ou baixos, a mediana representa o ponto central exacto onde 50% das observações ficam abaixo e 50% ficam acima.

Considere um conjunto de dados de 7 salários em uma pequena empresa: US$ 42.000, US$ 45.000, US$ 48.000, US$ 52.000, US$ 55.000, US$ 58.000 e US$ 320.000 (o CEO). O salário médio seria de US$ 88.571, o que representa erroneamente o que a maioria dos funcionários ganha. O salário médio de US$ 52.000 conta uma história mais precisa sobre a remuneração típica.

Esta calculadora lida com conjuntos de dados de contagem ímpar (onde existe um valor médio) e conjuntos de dados de contagem par (onde a mediana é a média dos dois valores médios). Esteja você analisando pontuações de testes, preços de moradias, tempos de resposta ou classificações de pesquisas, compreender a mediana ajuda a tomar decisões com base em dados representativos, em vez de médias distorcidas.

Fórmulas medianas com cálculos completos

Para conjuntos de dados ímpares (n é ímpar):

Mediana = x(n+1)/2

Onde x representa os valores classificados en é a contagem de observações.

Para conjuntos de dados pares (n é par):

Mediana = (xn/2 + x(n/2)+1) / 2

A mediana é a média aritmética dos dois valores centrais.

5 exemplos detalhados

Exemplo 1: Preços de imóveis

Um corretor de imóveis rastreia os preços de venda de 9 casas em um bairro: US$ 285.000, US$ 298.000, US$ 310.000, US$ 315.000, US$ 322.000, US$ 335.000, US$ 348.000, US$ 365.000, US$ 890.000

Dados classificados: n = 9 (ímpar)

Posição intermediária: (9+1)/2 = 5

Mediana = US$ 322.000

A média seria de US$ 385.333, inflacionada pelo valor discrepante de US$ 890.000. A mediana de $ 322.000 representa melhor os valores residenciais típicos nesta área.

Exemplo 2: Tempos de resposta do atendimento ao cliente

Tempos de resposta (em minutos) para 12 tickets de suporte: 8, 12, 15, 18, 22, 25, 28, 32, 38, 45, 52, 180

Dados classificados: n = 12 (par)

Posições intermediárias: 12/2 = 6 e (12/2)+1 = 7

Valores: x₆ = 25, x₇ = 28

Mediana = (25 + 28) / 2 = 26,5 minutos

O valor atípico de 180 minutos (um ticket escalonado) aumentaria a média para 39,6 minutos, enganando a administração sobre o desempenho típico.

Exemplo 3: Pontuações dos testes dos alunos

Notas dos exames de 15 alunos: 62, 67, 71, 73, 75, 78, 79, 81, 82, 84, 86, 88, 89, 91, 94

Dados classificados: n = 15 (ímpar)

Posição intermediária: (15+1)/2 = 8

Mediana = 81

Metade da turma pontuou abaixo de 81, metade pontuou acima. Esta tendência central ajuda o professor a compreender o desempenho típico sem distorção das poucas pontuações muito baixas ou muito altas.

Exemplo 4: Taxas de defeitos de fabricação

Defeitos por 1.000 unidades em 8 turnos de produção: 12, 15, 17, 19, 21, 24, 28, 47

Dados classificados: n = 8 (par)

Posições intermediárias: 8/2 = 4 e (8/2)+1 = 5

Valores: x₄ = 19, x₅ = 21

Mediana = (19 + 21) / 2 = 20 defeitos por 1.000 unidades

Os 47 defeitos do turno 8 (mau funcionamento do equipamento) distorceriam a média para 22,9, exagerando as taxas típicas de defeitos.

Exemplo 5: Latência de download do aplicativo

Tempos de download (segundos) para 11 usuários: 2,3, 2,7, 3,1, 3,4, 3,8, 4,2, 4,6, 5,1, 5,8, 6,4, 12,7

Dados classificados: n = 11 (ímpar)

Posição intermediária: (11+1)/2 = 6

Mediana = 4,2 segundos

O valor discrepante de 12,7 segundos (usuário em conexão lenta) aumentaria a média para 4,9 segundos. A mediana de 4,2 segundos representa melhor a experiência típica do usuário.

4 erros comuns a evitar

Erro 1 – Ignorando a etapa de classificação:O erro mais frequente é encontrar o valor do meio em uma lista não classificada. Se seus dados forem [45, 12, 38, 29, 51], a posição intermediária é 38, mas a mediana correta é 38 somente após a classificação para [12, 29, 38, 45, 51]. Sempre classifique primeiro.

Erro 2 – Calculando a média de todos os valores em vez do meio dois:Para conjuntos de dados de contagem par, algumas pessoas calculam a média de todos os números, em vez de apenas os dois valores intermediários. Com dados [10, 20, 30, 40], a mediana é (20+30)/2 = 25, não (10+20+30+40)/4 = 25 (coincidentemente a mesma aqui, mas não geralmente).

Erro 3 – Posições de contagem incorreta:Erros ocasionais acontecem ao identificar posições intermediárias. Para n=11, o meio é a posição 6, não 5 ou 7. Escreva a fórmula: (n+1)/2 para ímpar, n/2 e (n/2)+1 para par.

Erro 4 — Incluir dados não numéricos ou inválidos:Entradas de texto, células em branco ou valores de espaço reservado como -999 corrompem cálculos medianos. Limpe seu conjunto de dados primeiro. Para [15, 18, "N/A", 22, 25], exclua a entrada de texto e calcule a mediana de [15, 18, 22, 25] = 20.

4 dicas práticas

Dica 1 — Use a mediana quando os dados estiverem distorcidos:As distribuições de renda, os preços das casas e os tempos de resposta normalmente têm caudas longas para a direita. A mediana fornece um valor “típico” mais representativo do que a média. Relate ambos para mostrar a magnitude da inclinação.

Dica 2 — Compare a mediana com a média para detecção de assimetria:Se mediana < média, os dados apresentam distorção positiva (altos valores discrepantes). Se mediana > média, os dados terão distorção negativa (valores discrepantes baixos). Se mediana ≈ média, os dados são aproximadamente simétricos.

Dica 3 — Para grandes conjuntos de dados, use software:Classificar manualmente 10.000 valores é impraticável. Use a função MEDIAN do Excel, Statistics.median() do Python ou a função median() do R. Eles lidam com classificação e casos extremos automaticamente.

Dica 4 — Relate o tamanho da amostra com mediana:Uma mediana baseada em 5 observações carrega menos confiança do que uma baseada em 500. Sempre indique n ao lado do valor mediano, especialmente em relatórios ou apresentações.

4 FAQs

Use a mediana quando seus dados contiverem valores discrepantes ou estiverem distorcidos. Os dados salariais, os preços das casas e os tempos de resposta normalmente têm valores extremos que distorcem a média. A mediana permanece estável independentemente de quão extremos os valores discrepantes se tornem. Para dados simétricos e normalmente distribuídos sem valores discrepantes, a média e a mediana serão semelhantes e ambas funcionam.

Sim, para conjuntos de dados de contagem par. Se seus dados classificados forem [10, 20, 30, 40], a mediana será (20+30)/2 = 25, que não aparece nos dados originais. Isto é matematicamente correto – a mediana representa o ponto central, não necessariamente uma observação real.

As duplicatas são tratadas como observações separadas e incluídas na ordem de classificação. Para [5, 7, 7, 7, 9], a mediana é 7 (o valor do meio). Para [5, 7, 7, 9], a mediana é (7+7)/2 = 7. As duplicatas não requerem tratamento especial — basta classificar e aplicar a fórmula padrão.

A mediana é simplesmente esse valor. Para [3, 8, 8, 8, 8, 12], os dois valores intermediários são ambos 8, então a mediana é (8+8)/2 = 8. Isso é comum em dados discretos com valores possíveis limitados, como respostas em escala Likert ou dados de contagem.
