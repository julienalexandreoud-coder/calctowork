O que é um gerador de números aleatórios?

Um gerador de números aleatórios (RNG) produz valores imprevisíveis dentro de um intervalo especificado, servindo funções críticas em criptografia, simulações científicas, jogos, amostragem estatística e tomada de decisões. Ao contrário das escolhas “aleatórias” geradas por humanos – que exibem preconceitos previsíveis – os RNGs adequados fornecem distribuições matematicamente uniformes, onde cada número tem igual probabilidade de seleção.

A diferença entre a verdadeira aleatoriedade e a aproximação humana é gritante. Peça a alguém para escolher um número aleatório entre 1 e 10, e 7 aparece 28% das vezes (vs. 10% esperado). As pessoas evitam repetir números, criam padrões falsos e preferem determinados dígitos. Num estudo de 8,5 milhões de sequências "aleatórias" geradas por humanos, os investigadores identificaram padrões previsíveis que permitem uma precisão de previsão de 15-27% - impossível com a verdadeira aleatoriedade, onde cada resultado é independente.

Existem dois tipos de RNGs: geradores de números aleatórios verdadeiros (TRNG) que coletam entropia de fenômenos físicos como ruído atmosférico, decaimento radioativo ou variações térmicas. Os geradores de números pseudoaleatórios (PRNG) usam algoritmos matemáticos a partir de um valor inicial. Os computadores modernos combinam ambos: PRNGs para velocidade em simulações e jogos, semeados por fontes TRNG para segurança criptográfica. Entender qual tipo seu aplicativo precisa evita erros dispendiosos — usar PRNGs previsíveis para criptografia ou exploração de jogos de azar.

Fórmula de geração de números aleatórios explicada

Fórmula do Gerador Congruente Linear (LCG):

Xₙ₊₁ = (a × Xₙ + c) mod m

Where:

  • Xₙ = Estado atual (semente ou valor anterior)
  • Xₙ₊₁ = Próximo valor aleatório
  • a = Constante multiplicadora
  • c = Constante de incremento
  • m = Módulo (define o intervalo de 0 a m-1)

Parâmetros LCG comuns (MINSTD):

  • a = 48.271
  • c = 0
  • m = 2.147.483.647 (2³¹ - 1, um primo de Mersenne)
  • Período: 2,1 bilhões de valores antes de repetir

Escalando para intervalo arbitrário [min, max]:

Aleatório = min + (Xₙ / m) × (máx - min + 1)

Ou usando módulo para intervalos inteiros:

Aleatório = min + (Xₙ mod (máx - min + 1))

Exemplo resolvido 1 - Gere o número aleatório 1-100:

Etapa 1: Valor inicial X₀ = 12.345 (estado inicial)

Etapa 2: Aplicar a fórmula LCG com parâmetros MINSTD:

X₁ = (48.271 × 12.345 + 0) módulo 2.147.483.647

X₁ = 595.904.595 mod 2.147.483.647 = 595.904.595

Etapa 3: dimensionar para o intervalo [1, 100]:

Aleatório = 1 + (595.904.595 mod 100) = 1 + 95 = 96

Resultado: 96(primeiro número aleatório no intervalo 1-100)

Etapa 4: Gere o próximo valor:

X₂ = (48.271 × 595.904.595) mod 2.147.483.647 = 1.308.818.422

Aleatório = 1 + (1.308.818.422 mod 100) = 1 + 22 = 23

Resultado: 23(segundo número aleatório)

Exemplo resolvido 2 - Byte aleatório criptográfico (0-255):

Os RNGs criptográficos usam fontes de entropia de hardware, não fórmulas LCG.

Etapa 1: Coletar entropia do sistema operacional (interrupções de tempo, ruído de hardware)

Etapa 2: alimentar a função hash criptográfica (SHA-256)

Etapa 3: Extraia o primeiro byte: 0x7F = 127 em decimal

Resultado: 127(imprevisível, adequado para chaves de criptografia)

Cálculos de probabilidade:

Para distribuição uniforme no intervalo [min, max]:

P(qualquer valor específico) = 1 / (máx - min + 1)

Para intervalo 1-100: P(qualquer número) = 1/100 = 0,01 = 1%

Valor esperado (média):

E[X] = (mín + máx) / 2

Para intervalo 1-100: E[X] = (1 + 100) / 2 = 50,5

6 etapas para gerar números aleatórios

  1. Defina o intervalo e o tipo de valores aleatórios necessários:Especifique os valores mínimo e máximo e se você precisa de números inteiros ou decimais. Lançando um dado: intervalo [1, 6], inteiros. Selecionando um número de loteria: intervalo [1, 69], inteiros. Chave criptográfica: intervalo [0, 255], bytes. Simulação científica: faixa [0,0, 1,0], ponto flutuante. A faixa determina o módulo e a abordagem de escala.
  2. Escolha o tipo de RNG apropriado para seu caso de uso:Para jogos e simulações: use PRNG (Mersenne Twister, PCG) para velocidade e reprodutibilidade. Para criptografia e segurança: use CSPRNG (crypto.getRandomValues, /dev/urandom) propagado por entropia de hardware. Para pesquisas científicas que exigem verdadeira aleatoriedade: use os serviços TRNG (random.org) baseados em ruído atmosférico. Nunca use PRNG padrão para senhas, criptografia ou jogos de azar.
  3. Inicialize (semeie) o gerador corretamente:PRNGs exigem um valor inicial – a mesma semente sempre produz a mesma sequência. Para reprodutibilidade (depuração, artigos científicos): use uma semente fixa como 42 ou 12345. Para imprevisibilidade (jogos, segurança): semente de fontes de alta entropia, como hora atual em microssegundos, ID do processo e contadores de hardware. RNGs criptográficos são propagados automaticamente a partir de pools de entropia do sistema operacional.
  4. Gere o valor aleatório bruto:Chame a função RNG para produzir um valor. Em JavaScript: Math.random() fornece [0,0, 1,0). Em Python: random.randint(1, 100) fornece números inteiros. Em C#: Random.Next(1, 101). Para criptografia: window.crypto.getRandomValues() preenche um array digitado com bytes aleatórios. A saída bruta pode precisar ser dimensionada para o seu intervalo alvo.
  5. Dimensione e transforme de acordo com seu intervalo alvo:Converta a saída bruta no intervalo desejado. Para números inteiros [min, max]: resultado = min + (mod bruto (max - min + 1)). Para carros flutuantes [min, max]: resultado = min + bruto × (max - min). Garanta uma distribuição uniforme – o dimensionamento ingênuo pode introduzir preconceitos. Por exemplo, gerar 1-10 de 1-100 dividindo por 10 cria resultados não uniformes; use amostragem de módulo ou rejeição.
  6. Verifique propriedades estatísticas para aplicações críticas:Execute testes qui-quadrado para confirmar a distribuição uniforme. Gere mais de 10.000 amostras e verifique se cada valor aparece aproximadamente igualmente. Para uso criptográfico, aplique o NIST Statistical Test Suite (STS) validando frequência, execuções e propriedades de autocorrelação. Testes com falha indicam saída tendenciosa que requer um algoritmo RNG ou fonte de semente diferente.

5 exemplos práticos de geração de números aleatórios

Exemplo 1 — Lançamento de Dados do Jogo de Tabuleiro (2d6):

  • Alcance por dado: [1, 6]
  • Tipo de RNG: PRNG aceitável (sem requisitos de segurança)
  • Gere o primeiro dado: random(1, 6) = 4
  • Gere o segundo dado: random(1, 6) = 5
  • Total: 4 + 5 = 9
  • Result: 9(rolagem de dados típica para Monopoly ou Catan)

Exemplo 2 — Seleção de número de loteria (6 números de 1 a 49):

  • Intervalo: [1, 49], nenhuma repetição permitida
  • Tipo RNG: PRNG com amostragem de rejeição
  • Gere: 17, 8, 33, 42, 5, 29 (rejeite quaisquer duplicatas)
  • Classificar para exibição: 5, 8, 17, 29, 33, 42
  • Resultado: [5, 8, 17, 29, 33, 42](escolha válida na loteria)
  • Probabilidade desta combinação exata: 1 em 13.983.816

Exemplo 3 — Atribuição de teste A/B (usuário para variante):

  • Faixa: [0, 1) float, atribuir <0,5 à variante A, ≥0,5 à variante B
  • Tipo RNG: PRNG (reprodutibilidade útil para depuração)
  • Gerar: random() = 0,73294
  • Atribuição: 0,73294 ≥ 0,5, então Variante B
  • Resultado: Usuário atribuído à Variante B(divisão 50/50)
  • Mais de 10.000 usuários: espere aproximadamente 5.000 por variante (±100 por aleatoriedade)

Exemplo 4 — Geração de chave de criptografia (128 bits):

  • Intervalo: [0, 255] para cada um dos 16 bytes
  • Tipo de RNG: CSPRNG necessário (crítico para segurança)
  • Gere 16 bytes aleatórios: 0x3A, 0xF7, 0x2C, 0x8D, 0x14, 0xE9, 0x76, 0xB3, 0x45, 0x9A, 0x0F, 0xC8, 0x51, 0xD2, 0x8E, 0x67
  • Representação hexadecimal: 3AF72C8D14E976B3459A0FC851D28E67
  • Resultado: chave de 128 bits com 2¹²⁸ (3,4 × 10³⁸) valores possíveis
  • Tempo de cracking: computacionalmente inviável com a tecnologia atual

Exemplo 5 — Simulação de Monte Carlo (estimativa de π):

  • Gere pontos aleatórios (x, y) no quadrado unitário [0, 1] × [0, 1]
  • Conte os pontos onde x² + y² ≤ 1 (dentro do quarto de círculo)
  • Gere 10.000 pontos: 7.854 caem dentro do círculo
  • π ≈ 4 × (dentro / total) = 4 × (7.854 / 10.000) = 3,1416
  • Resultado: π ≈ 3,1416(π real = 3,14159..., erro = 0,003%)
  • Mais amostras = melhor precisão (lei dos grandes números)

4 erros comuns de geração de números aleatórios

  • Usando RNG não criptográfico para fins de segurança:Math.random() do JavaScript, módulo aleatório do Python e rand() do C usam algoritmos PRNG previsíveis. Um invasor observando 624 saídas consecutivas do Mersenne Twister pode prever todos os valores futuros. Nunca use-os para geração de senhas, tokens de sessão ou chaves de criptografia. Sempre use crypto.getRandomValues() (navegadores), módulo de segredos (Python) ou /dev/urandom (Linux) para aleatoriedade sensível à segurança.
  • Introduzindo viés por meio de escala de intervalo inadequada:Gerar aleatoriamente 1-10 de random(1, 100) por cálculo (random(1, 100)/10) + 1 cria uma distribuição não uniforme. Os valores de 1 a 9 podem resultar de 10 entradas cada, mas 10 resulta apenas de 91 a 100 (10 entradas) — espere, este exemplo é realmente uniforme. Melhor exemplo: random(1, 7) mod 5 + 1 fornece 1-3 de 2 entradas cada, mas 4-5 de 1 entrada cada, tendendo para números mais baixos. Use amostragem de rejeição: gere até que o valor caia em uma faixa uniforme.
  • Semeando com valores previsíveis:Usar o carimbo de data/hora atual (segundos) como semente significa que os invasores podem reduzir a semente para aproximadamente 86.400 possibilidades por dia. Se eles souberem aproximadamente quando você gerou um valor, eles poderão forçar a semente e prever todas as saídas. Use sementes de alta entropia: combine carimbo de data/hora com ID de processo, contadores de hardware e entropia de sistema operacional. Para criptografia, nunca propague manualmente — deixe o CSPRNG lidar com a propagação internamente.
  • Assumir aleatoriedade significa "sem padrões":Os humanos esperam que sequências aleatórias se alternem com frequência (HTHTHT para lançamentos de moeda), mas a verdadeira aleatoriedade produz agrupamentos (HHHHHT, TTTTTH). Uma sequência de dez 7 consecutivos tem probabilidade de 1 em 10¹⁰ – improvável, mas garantida de ocorrer eventualmente em tentativas infinitas. Os RNGs que produzem distribuições "muito uniformes" (evitando clusters) são, na verdade, tendenciosos. Os testes estatísticos verificam as propriedades esperadas de aleatoriedade, e não a ausência de padrões.

5 dicas profissionais para geração de números aleatórios

  • Use o Mersenne Twister para simulações científicas:MT19337 tem um período de 2¹⁹⁹³⁷ - 1 (um número com 6.002 dígitos), passa em todos os testes estatísticos padrão e gera valores 3-10× mais rápido que RNGs criptográficos. É o padrão em Python, R e NumPy por um bom motivo. Para reprodutibilidade, armazene o valor inicial em suas notas de pesquisa. Advertência: MT é previsível após observar 624 resultados – nunca use para criptografia.
  • Implementar amostragem de rejeição para resultados imparciais:Ao gerar valores no intervalo [0, N) a partir de um intervalo maior [0, M), rejeite valores ≥ floor(M/N) × N. Exemplo: geração de 1-10 a partir do byte aleatório [0, 255]. Piso(256/10) × 10 = 250. Gere um byte; se valor ≥ 250, descarte e regenere. Isso garante que cada saída 0-9 tenha exatamente 25 valores de entrada mapeados para ela, eliminando a polarização do módulo. Um pouco mais lento, mas matematicamente correto.
  • Aproveite o RNG de hardware para entropia máxima:CPUs modernas incluem instruções RDRAND (Intel) ou RDSEED (AMD) que acessam fontes de entropia no chip. Em C/C++: use _rdrand32_step(). Em ferrugem: rand::rngs::OsRng. Em Go: crypto/rand.Read(). Eles se baseiam no ruído térmico, na instabilidade do clock e na metaestabilidade do silício – verdadeira aleatoriedade física. Mais lento que os PRNGs, mas essencial para a geração de chaves. Use uma abordagem híbrida: propague PRNG a partir de RNG de hardware para obter velocidade com segurança.
  • Teste a saída RNG com conjuntos de testes estatísticos:Para aplicações críticas, valide seu RNG usando NIST STS (15 testes para RNGs criptográficos), Dieharder (17 testes) ou TestU01 (bateria abrangente). Eles verificam a distribuição de frequência, execuções, autocorrelação e propriedades espectrais. Um teste reprovado indica viés que pode comprometer as simulações ou a segurança. Execute testes em amostras de mais de 1 megabit — amostras pequenas podem passar por acaso, mesmo com RNGs quebrados.
  • Documente os valores iniciais para reprodutibilidade:Na pesquisa científica, registre a semente exata e o algoritmo RNG usado. "Python 3.9, random.seed(42), Mersenne Twister" permite que outros pesquisadores reproduzam suas sequências aleatórias. Isso é essencial para depuração, revisão por pares e desenvolvimento de trabalhos anteriores. Para sistemas de produção, registre sementes usadas para cada sessão para permitir análise post-hoc de comportamento anômalo.

4 perguntas frequentes sobre números aleatórios

Math.random() usa um algoritmo PRNG rápido (geralmente xorshift128+) adequado para jogos e simulações, mas previsível se os resultados forem observados. crypto.getRandomValues() usa o RNG criptograficamente seguro do sistema operacional, propagado a partir da entropia de hardware, tornando as saídas imprevisíveis mesmo com pleno conhecimento dos valores anteriores. Use Math.random() para lançamentos de dados do jogo; use crypto.getRandomValues() para senhas, tokens e chaves de criptografia. Diferença de desempenho: ~100× mais rápido para Math.random() mas a diferença de segurança é infinita (previsível vs. imprevisível).

PRNGs são determinísticos – a mesma semente sempre produz a mesma sequência. A observação de resultados suficientes (624 para Mersenne Twister) permite prever todos os valores futuros. TRNGs e CSPRNGs propagados a partir da entropia de hardware são computacionalmente imprevisíveis – nenhum algoritmo conhecido pode prever a próxima saída a partir de valores anteriores sem conhecer o estado interno. No entanto, implementações falhas (o OpenSSL com bugs do Debian em 2008, reduzindo a entropia para 32.767 possibilidades) podem se tornar previsíveis. Sempre use bibliotecas RNG bem auditadas.

Seu RNG é propagado com um valor fixo (possivelmente padronizado como 0 ou 1). Muitos idiomas usam uma semente padrão para reprodutibilidade, caso você não especifique uma. Em Python, chame random.seed() sem argumentos para propagar a partir da hora do sistema. Em C++, use std::random_device para propagar. Em JavaScript, Math.random() é propagado automaticamente no carregamento da página. Para resultados reproduzíveis (testes, pesquisas), use intencionalmente uma semente fixa como random.seed(42). Para resultados imprevisíveis, garanta a propagação de fontes de alta entropia.

Esta é a verdadeira aleatoriedade (TRNG), não a pseudo-aleatoriedade algorítmica. Auditorias independentes confirmam a distribuição uniforme e a imprevisibilidade.
