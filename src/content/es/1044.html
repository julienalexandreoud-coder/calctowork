¿Qué es la relación de contraste?

La relación de contraste mide la diferencia de luminancia entre las partes más claras y más oscuras de una imagen o interfaz. Una relación de contraste de 21:1 representa el máximo contraste (blanco puro sobre negro puro), mientras que 1:1 representa ningún contraste (colores idénticos). Las Pautas de accesibilidad al contenido web (WCAG) requieren relaciones de contraste mínimas para el texto legible.

También puede encontrar elHex to RGB Converter, Decibel Addition Calculator, and Exposure Value Calculator useful.

Los diseñadores web utilizan relaciones de contraste para garantizar que el texto sea legible para usuarios con discapacidad visual. El texto negro (#000000) sobre fondo blanco (#FFFFFF) logra un contraste de 21:1, superando con creces el requisito WCAG AA de 4,5:1 para texto normal. El texto gris claro (#999999) sobre blanco alcanza solo 2,8:1, lo que incumple los estándares de accesibilidad y es difícil de leer para muchos usuarios.

Los desarrolladores de UI verifican las relaciones de contraste de elementos interactivos como botones y enlaces. Un botón azul (#4A90D9) con texto blanco logra un contraste de 4,6:1, superando las WCAG AA. El mismo azul con texto gris claro (#CCCCCC) cae a 1,8:1, lo que falla y es potencialmente inutilizable para usuarios con baja visión o daltonismo.

Fórmulas de relación de contraste con cálculos elaborados

WCAG 2.1 define la relación de contraste utilizando la luminancia relativa:

CR = (L₁ + 0,05) / (L₂ + 0,05)

Donde L₁ es la luminancia relativa del color más claro (0 a 1) y L₂ es la luminancia relativa del color más oscuro (0 a 1).

La luminancia relativa se calcula a partir de valores sRGB:

L = 0,2126 × R + 0,7152 × G + 0,0722 × B

Donde R, G y B son los valores de color linealizados (corregidos con gamma).

Ejemplo resuelto:Calcule el contraste entre #3366CC (azul) y #FFFFFF (blanco).

Primero, convierta hexadecimal a sRGB (escala 0-255): #3366CC = rgb(51, 102, 204), #FFFFFF = rgb(255, 255, 255).

Normalizar a escala 0-1: R = 51/255 = 0,2, G = 102/255 = 0,4, B = 204/255 = 0,8 para azul. El blanco es R = G = B = 1.

Linealizar cada canal (corrección gamma inversa):

Si valor ≤ 0,03928: lineal = valor / 12,92

Si valor > 0,03928: lineal = ((valor + 0,055) / 1,055)^2.4

Para azul: R = ((0,2 + 0,055) / 1,055)^2,4 = 0,0337, G = ((0,4 + 0,055) / 1,055)^2,4 = 0,1369, B = ((0,8 + 0,055) / 1,055)^2,4 = 0,6072

Para blanco: R = G = B = ((1 + 0,055) / 1,055)^2,4 = 1,0

Calcular la luminancia relativa:

L_azul = 0,2126 × 0,0337 + 0,7152 × 0,1369 + 0,0722 × 0,6072 = 0,0072 + 0,0979 + 0,0438 = 0,1489

L_blanco = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 1 = 1,0

Calcular la relación de contraste:

CR = (1,0 + 0,05) / (0,1489 + 0,05) = 1,05 / 0,1989 = 5,28:1

Resultado: Azul #3366CC sobre blanco #FFFFFF logra un contraste de 5.28:1: pasa WCAG AA (4.5:1) y AAA (3:1) para texto normal.

Cómo calcular la relación de contraste: 6 pasos

  1. Convertir colores a RGB:Su color de primer plano es #7B2D8E (púrpura) y el fondo es #F5F5F5 (gris claro). Convertir: #7B2D8E = rgb(123, 45, 142), #F5F5F5 = rgb(245, 245, 245).
  2. Normalizar a escala 0-1:Púrpura: R = 123/255 = 0,482, G = 45/255 = 0,176, B = 142/255 = 0,557. Gris: R = G = B = 245/255 = 0,961.
  3. Aplicar corrección gamma:For purple: R = ((0.482 + 0.055) / 1.055)^2.4 = 0.203, G = ((0.176 + 0.055) / 1.055)^2.4 = 0.026, B = ((0.557 + 0.055) / 1.055)^2.4 = 0.272. Para gris: R = G = B = ((0,961 + 0,055) / 1,055)^2,4 = 0,919.
  4. Calcular la luminancia relativa:L_púrpura = 0,2126 × 0,203 + 0,7152 × 0,026 + 0,0722 × 0,272 = 0,043 + 0,019 + 0,020 = 0,082. L_gris = 0,2126 × 0,919 + 0,7152 × 0,919 + 0,0722 × 0,919 = 0,919.
  5. Identificar más claro y más oscuro:L_gray (0,919) > L_purple (0,082), entonces L₁ = 0,919 y L₂ = 0,082.
  6. Aplicar fórmula de contraste:RC = (0,919 + 0,05) / (0,082 + 0,05) = 0,969 / 0,132 = 7,34:1. Este violeta sobre gris claro logra 7.34:1: pasa WCAG AAA (7:1) para texto normal.

5 ejemplos de relación de contraste

Ejemplo 1: negro sobre blanco (contraste máximo):#000000 en #FFFFFF. L_black = 0 (todos los canales se linealizan a 0). L_blanco = 1. CR = (1 + 0,05) / (0 + 0,05) = 1,05 / 0,05 = 21:1. Este es el máximo contraste posible en el espacio de color sRGB, el estándar de oro para la legibilidad.

Ejemplo 2: Gris sobre gris (contraste fallido):#999999 en #CCCCCC. L_luz = 0,2126 × 0,502 + 0,7152 × 0,502 + 0,0722 × 0,502 = 0,502. L_oscuro = 0,2126 × 0,282 + 0,7152 × 0,282 + 0,0722 × 0,282 = 0,282. CR = (0,502 + 0,05) / (0,282 + 0,05) = 0,552 / 0,332 = 1,66:1. Esto falla en todos los niveles WCAG, algo común en interfaces mal diseñadas donde el texto "sutil" gris sobre gris se vuelve ilegible.

Ejemplo 3: Verde sobre blanco (límite AA):#00AA00 (verde brillante) en #FFFFFF. L_verde = 0,2126 × 0 + 0,7152 × 0,331 + 0,0722 × 0 = 0,237. L_blanco = 1. CR = (1 + 0,05) / (0,237 + 0,05) = 1,05 / 0,287 = 3,66:1. Esto no supera la WCAG AA (4.5:1) para texto normal, pero pasa para texto grande (18 puntos o 14 puntos en negrita, requiriendo 3:1). El texto verde es particularmente problemático para los usuarios daltónicos rojo-verde.

Ejemplo 4: Amarillo sobre negro (alto contraste):#FFFF00 (amarillo puro) en #000000. L_amarillo = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 0 = 0,928. L_negro = 0. CR = (0,928 + 0,05) / (0 + 0,05) = 0,978 / 0,05 = 19,56:1. El amarillo sobre negro logra un contraste casi máximo: por eso las señales de advertencia y las etiquetas de peligro utilizan esta combinación para una máxima visibilidad.

Ejemplo 5: enlace azul sobre blanco (patrón web común):#0000EE (enlace estándar azul) en #FFFFFF. L_azul = 0,2126 × 0 + 0,7152 × 0,033 + 0,0722 × 0,549 = 0,024 + 0,040 = 0,064. L_blanco = 1. CR = (1 + 0,05) / (0,064 + 0,05) = 1,05 / 0,114 = 9,21:1. El enlace azul tradicional pasa fácilmente todos los niveles WCAG, una de las razones por las que se convirtió en el estándar web en la década de 1990 y sigue siendo accesible en la actualidad.

4 errores comunes en la relación de contraste

  • Usando fórmulas de luminancia simples:Calcular la luminancia como (R + G + B) / 3 ignora la diferente sensibilidad de la visión humana a cada color. El verde aporta el 71,52% del brillo percibido, el rojo el 21,26% y el azul sólo el 7,22%. Un color como #00FF00 (verde puro) parece mucho más brillante que #0000FF (azul puro) a pesar de tener sumas RGB iguales.
  • Saltar corrección gamma:El uso de valores sRGB directamente sin linealizar produce una luminancia incorrecta. El espacio de color sRGB utiliza una curva gamma para igualar la percepción humana. #808080 (gris medio) tiene una luminancia lineal de 0,216, no de 0,5, una diferencia del 57 % que afecta significativamente los cálculos de contraste.
  • Probando solo tamaños de texto normales:WCAG tiene diferentes requisitos para texto normal (4.5:1 AA, 7:1 AAA) versus texto grande (3:1 AA, 4.5:1 AAA). El texto grande está en negrita de 18 o 14 puntos. Un contraste de 3,5:1 no es AA para un cuerpo de texto de 12 puntos, pero sí para títulos de 20 puntos.
  • Ignorando elementos que no son de texto:WCAG también requiere un contraste 3:1 para los componentes de la interfaz de usuario (botones, bordes de entrada) y objetos gráficos (iconos, gráficos). Un borde de botón gris (#CCCCCC) sobre fondo blanco (#FFFFFF) logra solo 1.3:1, incumpliendo el requisito de 3:1 y haciendo que el botón sea invisible para muchos usuarios.

Cinco consejos para cumplir con la relación de contraste

  • Comience con valores predeterminados de alto contraste:Utilice #000000 en #FFFFFF (21:1) o #1A1A1A en #FAFAFA (18:1) como base. Estos superan todos los requisitos y ofrecen margen para variaciones de diseño. Solo reduzca el contraste intencionalmente para elementos de diseño específicos, nunca para el cuerpo del texto.
  • Utilice las herramientas de desarrollo del navegador:Chrome y Firefox DevTools muestran relaciones de contraste en el selector de color. Haga clic en cualquier color en el panel Estilos y el selector mostrará la relación de contraste contra el fondo. También muestra aprobación/rechazo para WCAG AA y AAA de un vistazo.
  • Prueba con simuladores de daltonismo:Herramientas como Stark, Color Oracle o la configuración de renderizado de Chrome simulan varios tipos de deficiencia en la visión del color. Un contraste rojo-verde que pasa matemáticamente (4,5:1) aún puede resultar ilegible para los usuarios deuteranópicos si el tono es el único factor distintivo.
  • Verifique los estados de enfoque y desplazamiento:Un botón con contraste 5:1 en su estado predeterminado puede caer a 2:1 al pasar el mouse si el fondo se aclara. Pruebe todos los estados interactivos: predeterminado, flotante, enfocado, activo y deshabilitado. Cada estado debe cumplir con requisitos mínimos de contraste.
  • Utilice herramientas de auditoría automatizadas:Lighthouse (integrado en Chrome), ax DevTools o WAVE pueden escanear páginas enteras en busca de violaciones de contraste. Estas herramientas detectan problemas que las pruebas manuales pasan por alto, como contenido generado dinámicamente o variables CSS que cambian el contraste según el tema.

El nivel AAA requiere 7:1 para texto normal, 4,5:1 para texto grande y 4,5:1 para componentes de interfaz de usuario. La constante de 0,05 tiene en cuenta el brillo de la pantalla y los reflejos de la luz ambiental. En condiciones de visualización del mundo real, incluso el "negro puro" en una pantalla refleja algo de luz, y el "blanco puro" no parece infinitamente brillante. La compensación evita relaciones de contraste infinitas (dividiendo por cero) y modela entornos de visualización típicos con luz ambiental moderada.
