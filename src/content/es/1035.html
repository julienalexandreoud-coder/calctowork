¿Qué es un generador de números aleatorios?

Un generador de números aleatorios (RNG) produce valores impredecibles dentro de un rango específico y cumple funciones críticas en criptografía, simulaciones científicas, juegos, muestreo estadístico y toma de decisiones. A diferencia de las elecciones "aleatorias" generadas por humanos, que exhiben sesgos predecibles, los RNG adecuados ofrecen distribuciones matemáticamente uniformes donde cada número tiene la misma probabilidad de selección.

La diferencia entre la verdadera aleatoriedad y la aproximación humana es marcada. Pídale a alguien que elija un número aleatorio entre 1 y 10, y el 7 aparece el 28 % de las veces (frente al 10 % esperado). La gente evita repetir números, crea patrones falsos y prefiere ciertos dígitos. En un estudio de 8,5 millones de secuencias "aleatorias" generadas por humanos, los investigadores identificaron patrones predecibles que permitían una precisión de predicción del 15 al 27%, algo imposible con una verdadera aleatoriedad donde cada resultado es independiente.

Existen dos tipos de RNG: los verdaderos generadores de números aleatorios (TRNG) recolectan entropía de fenómenos físicos como el ruido atmosférico, la desintegración radiactiva o las variaciones térmicas. Los generadores de números pseudoaleatorios (PRNG) utilizan algoritmos matemáticos a partir de un valor inicial. Las computadoras modernas combinan ambos: PRNG para velocidad en simulaciones y juegos, sembrados con fuentes TRNG para seguridad criptográfica. Comprender qué tipo necesita su aplicación evita errores costosos: el uso de PRNG predecibles para el cifrado o los juegos de azar invita a la explotación.

Fórmula de generación de números aleatorios explicada

Fórmula del generador lineal congruencial (LCG):

Xₙ₊₁ = (a × Xₙ + c) mod m

Where:

  • Xₙ = Estado actual (valor inicial o anterior)
  • Xₙ₊₁ = Siguiente valor aleatorio
  • a = Constante multiplicadora
  • c = Constante incremental
  • m = Módulo (define el rango de 0 a m-1)

Parámetros LCG comunes (MINSTD):

  • a = 48,271
  • c = 0
  • m = 2.147.483.647 (2³¹ - 1, un primo de Mersenne)
  • Período: 2,1 mil millones de valores antes de repetir

Escalado a rango arbitrario [min, max]:

Aleatorio = min + (Xₙ / m) × (max - min + 1)

O usando módulo para rangos de números enteros:

Aleatorio = min + (Xₙ mod (max - min + 1))

Ejemplo resuelto 1: generar un número aleatorio del 1 al 100:

Paso 1: Valor de semilla X₀ = 12,345 (estado inicial)

Paso 2: Aplicar la fórmula LCG con parámetros MINSTD:

X₁ = (48,271 × 12,345 + 0) mod 2,147,483,647

X₁ = 595.904.595 mod 2.147.483.647 = 595.904.595

Paso 3: Escalar al rango [1, 100]:

Aleatorio = 1 + (595,904,595 mod 100) = 1 + 95 = 96

Resultado: 96(primer número aleatorio en el rango 1-100)

Paso 4: Generar el siguiente valor:

X₂ = (48,271 × 595,904,595) mod 2,147,483,647 = 1,308,818,422

Aleatorio = 1 + (1.308.818.422 mod 100) = 1 + 22 = 23

Resultado: 23(segundo número aleatorio)

Los RNG criptográficos utilizan fuentes de entropía de hardware, no fórmulas LCG.

Paso 1: recopilar entropía del sistema operativo (interrupciones de sincronización, ruido de hardware)

Paso 2: Introducir la función hash criptográfica (SHA-256)

Paso 3: Extraiga el primer byte: 0x7F = 127 en decimal

Resultado: 127(impredecible, adecuado para claves de cifrado)

Cálculos de probabilidad:

Para una distribución uniforme en el rango [min, max]:

P(cualquier valor específico) = 1 / (max - min + 1)

Para el rango 1-100: P(cualquier número) = 1/100 = 0,01 = 1%

Valor esperado (media):

E[X] = (mín + máx) / 2

Para el rango 1-100: E[X] = (1 + 100) / 2 = 50,5

6 pasos para generar números aleatorios

  1. Defina el rango y tipo de valores aleatorios necesarios:Especifique valores mínimos y máximos, y si necesita números enteros o decimales. Lanzar un dado: rango [1, 6], números enteros. Seleccionar un número de lotería: rango [1, 69], números enteros. Clave criptográfica: rango [0, 255], bytes. Simulación científica: rango [0,0, 1,0], punto flotante. El rango determina el módulo y el enfoque de escala.
  2. Elija el tipo de RNG apropiado para su caso de uso:Para juegos y simulaciones: utilice PRNG (Mersenne Twister, PCG) para mayor velocidad y reproducibilidad. Para criptografía y seguridad: utilice CSPRNG (crypto.getRandomValues, /dev/urandom) sembrado por entropía de hardware. Para investigaciones científicas que requieran verdadera aleatoriedad: utilice los servicios TRNG (random.org) basados ​​en el ruido atmosférico. Nunca utilice PRNG estándar para contraseñas, cifrado o juegos de apuestas.
  3. Inicialice (inicie) el generador correctamente:Los PRNG requieren un valor inicial: la misma semilla siempre produce la misma secuencia. Para reproducibilidad (depuración, artículos científicos): use una semilla fija como 42 o 12345. Para imprevisibilidad (juegos, seguridad): semilla de fuentes de alta entropía como la hora actual en microsegundos, ID de proceso y contadores de hardware. Los RNG criptográficos se generan automáticamente a partir de grupos de entropía del sistema operativo.
  4. Genere el valor aleatorio sin procesar:Llame a la función RNG para producir un valor. En JavaScript: Math.random() da [0.0, 1.0). En Python: random.randint(1, 100) da números enteros. En C#: Aleatorio.Siguiente(1, 101). Para criptografía: window.crypto.getRandomValues() llena una matriz escrita con bytes aleatorios. Es posible que sea necesario escalar la salida sin procesar a su rango objetivo.
  5. Escale y transforme a su rango objetivo:Convierta la salida sin procesar al rango deseado. Para números enteros [min, max]: resultado = min + (mod sin procesar (max - min + 1)). Para flotadores [min, max]: resultado = min + raw × (max - min). Garantice una distribución uniforme: una escala ingenua puede introducir sesgos. Por ejemplo, generar 1-10 a partir de 1-100 dividiendo por 10 crea resultados no uniformes; utilice muestreo de módulo o rechazo en su lugar.
  6. Verificar propiedades estadísticas para aplicaciones críticas:Ejecute pruebas de chi-cuadrado para confirmar una distribución uniforme. Genere más de 10 000 muestras y verifique que cada valor aparezca aproximadamente de la misma manera. Para uso criptográfico, aplique la frecuencia de validación, las ejecuciones y las propiedades de autocorrelación de NIST Statistical Test Suite (STS). Las pruebas fallidas indican una salida sesgada que requiere un algoritmo RNG o una fuente semilla diferente.

5 ejemplos prácticos de generación de números aleatorios

Ejemplo 1: Tirada de dados de un juego de mesa (2d6):

  • Rango por dado: [1, 6]
  • Tipo de RNG: PRNG aceptable (sin requisitos de seguridad)
  • Generar el primer dado: aleatorio(1, 6) = 4
  • Generar segundo dado: aleatorio(1, 6) = 5
  • Total: 4 + 5 = 9
  • Result: 9(tirada de dados típica de Monopoly o Catan)

Ejemplo 2: Selección de números de lotería (6 números del 1 al 49):

  • Rango: [1, 49], no se permiten repeticiones
  • Tipo RNG: PRNG con muestreo de rechazo
  • Generar: 17, 8, 33, 42, 5, 29 (rechazar cualquier duplicado)
  • Ordenar para visualización: 5, 8, 17, 29, 33, 42
  • Resultado: [5, 8, 17, 29, 33, 42](selección de lotería válida)
  • Probabilidad de esta combinación exacta: 1 en 13.983.816

Ejemplo 3: Asignación de prueba A/B (usuario a variante):

  • Rango: [0, 1) flotante, asignar <0,5 a la variante A, ≥0,5 a la variante B
  • Tipo de RNG: PRNG (reproducibilidad útil para depuración)
  • Generar: aleatorio() = 0.73294
  • Asignación: 0,73294 ≥ 0,5, por lo que Variante B
  • Resultado: Usuario asignado a la variante B(división 50/50)
  • Más de 10.000 usuarios: espere ~5.000 por variante (±100 por aleatoriedad)

Ejemplo 4: Generación de clave de cifrado (128 bits):

  • Rango: [0, 255] para cada uno de los 16 bytes
  • Tipo de RNG: se requiere CSPRNG (crítico para la seguridad)
  • Genera 16 bytes aleatorios: 0x3A, 0xF7, 0x2C, 0x8D, 0x14, 0xE9, 0x76, 0xB3, 0x45, 0x9A, 0x0F, 0xC8, 0x51, 0xD2, 0x8E, 0x67
  • Representación hexadecimal: 3AF72C8D14E976B3459A0FC851D28E67
  • Resultado: clave de 128 bits con 2¹²⁸ (3,4 × 10³⁸) valores posibles
  • Tiempo de craqueo: computacionalmente inviable con la tecnología actual

Ejemplo 5: simulación de Monte Carlo (estimación de π):

  • Genera puntos aleatorios (x, y) en el cuadrado unitario [0, 1] × [0, 1]
  • Cuente los puntos donde x² + y² ≤ 1 (dentro del cuarto de círculo)
  • Genera 10.000 puntos: 7.854 caen dentro del círculo
  • π ≈ 4 × (dentro / total) = 4 × (7.854 / 10.000) = 3,1416
  • Resultado: π ≈ 3.1416(π real = 3,14159..., error = 0,003%)
  • Más muestras = mejor precisión (ley de los grandes números)

4 errores comunes en la generación de números aleatorios

  • Uso de RNG no criptográfico por motivos de seguridad:Math.random() de JavaScript, el módulo aleatorio de Python y rand() de C utilizan algoritmos PRNG predecibles. Un atacante que observe 624 salidas consecutivas de Mersenne Twister puede predecir todos los valores futuros. Nunca los utilice para generar contraseñas, tokens de sesión o claves de cifrado. Utilice siempre crypto.getRandomValues() (navegadores), módulo de secretos (Python) o /dev/urandom (Linux) para aleatoriedad sensible a la seguridad.
  • Introduciendo sesgos a través de una escala de rango incorrecta:Generar aleatorio 1-10 a partir de aleatorio (1, 100) calculando (aleatorio (1, 100) / 10) + 1 crea una distribución no uniforme. Los valores del 1 al 9 pueden resultar de 10 entradas cada uno, pero 10 solo resultan del 91 al 100 (10 entradas). Espere, este ejemplo es realmente uniforme. Mejor ejemplo: random(1, 7) mod 5 + 1 da 1-3 de 2 entradas cada uno, pero 4-5 de 1 entrada cada uno, sesgando hacia números más bajos. Utilice muestreo de rechazo: genere hasta que el valor caiga en un rango uniforme.
  • Siembra con valores predecibles:Usar la marca de tiempo actual (segundos) como semilla significa que los atacantes pueden limitar la semilla a ~86,400 posibilidades por día. Si saben aproximadamente cuándo generó un valor, pueden aplicar fuerza bruta a la semilla y predecir todos los resultados. Utilice semillas de alta entropía: combine la marca de tiempo con la identificación del proceso, los contadores de hardware y la entropía del sistema operativo. Para la criptografía, nunca realice la siembra manualmente; deje que CSPRNG se encargue de la siembra internamente.
  • Asumir aleatoriedad significa "sin patrones":Los humanos esperan que las secuencias aleatorias se alternen con frecuencia (HTHTHT para lanzar monedas), pero la verdadera aleatoriedad produce grupos (HHHHHT, TTTTTH). Una secuencia de diez 7 seguidos tiene una probabilidad de 1 entre 10¹⁰: es poco probable, pero se garantiza que eventualmente ocurrirá en infinitas pruebas. Los RNG que producen distribuciones "demasiado uniformes" (evitando grupos) en realidad están sesgados. Las pruebas estadísticas verifican las propiedades de aleatoriedad esperadas, no la ausencia de patrones.

5 consejos profesionales para la generación de números aleatorios

  • Utilice el Mersenne Twister para simulaciones científicas:MT19337 tiene un período de 2¹⁹⁹³⁷ - 1 (un número con 6002 dígitos), pasa todas las pruebas estadísticas estándar y genera valores entre 3 y 10 veces más rápido que los RNG criptográficos. Es el valor predeterminado en Python, R y NumPy por una buena razón. Para mayor reproducibilidad, almacene el valor de la semilla en sus notas de investigación. Advertencia: MT es predecible después de observar 624 resultados; nunca lo use para criptografía.
  • Implementar muestreo de rechazo para obtener resultados imparciales:Al generar valores en el rango [0, N) desde un rango mayor [0, M), rechace los valores ≥ piso (M/N) × N. Ejemplo: generar 1-10 a partir del byte aleatorio [0, 255]. Piso(256/10) × 10 = 250. Generar un byte; si el valor es ≥ 250, descartar y regenerar. Esto garantiza que cada salida 0-9 tenga exactamente 25 valores de entrada asignados, eliminando el sesgo de módulo. Un poco más lento pero matemáticamente correcto.
  • Aproveche el RNG del hardware para obtener la máxima entropía:Las CPU modernas incluyen instrucciones RDRAND (Intel) o RDSEED (AMD) que acceden a fuentes de entropía en el chip. En C/C++: utilice _rdrand32_step(). En Rust: rand::rngs::OsRng. En Go: cripto/rand.Read(). Estos se basan en el ruido térmico, la fluctuación del reloj y la metaestabilidad del silicio: verdadera aleatoriedad física. Más lento que los PRNG pero esencial para la generación de claves. Utilice un enfoque híbrido: simule PRNG a partir de RNG de hardware para obtener velocidad y seguridad.
  • Pruebe la salida RNG con conjuntos de pruebas estadísticas:Para aplicaciones críticas, valide su RNG utilizando NIST STS (15 pruebas para RNG criptográficos), Dieharder (17 pruebas) o TestU01 (batería completa). Estos verifican la distribución de frecuencia, las ejecuciones, la autocorrelación y las propiedades espectrales. Una prueba fallida indica un sesgo que podría comprometer las simulaciones o la seguridad. Realice pruebas en muestras de más de 1 megabit: las muestras pequeñas pueden pasar por casualidad incluso con RNG rotos.
  • Documentar los valores de semilla para reproducibilidad:En la investigación científica, registre la semilla exacta y el algoritmo RNG utilizado. "Python 3.9, random.seed(42), Mersenne Twister" permite a otros investigadores reproducir sus secuencias aleatorias. Esto es esencial para la depuración, la revisión por pares y la elaboración de trabajos anteriores. Para los sistemas de producción, registre las semillas utilizadas en cada sesión para permitir el análisis post hoc del comportamiento anómalo.

4 preguntas frecuentes sobre números aleatorios

Math.random() utiliza un algoritmo PRNG rápido (a menudo xorshift128+) adecuado para juegos y simulaciones pero predecible si se observan los resultados. crypto.getRandomValues() utiliza el RNG criptográficamente seguro del sistema operativo, generado a partir de la entropía del hardware, lo que hace que las salidas sean impredecibles incluso con pleno conocimiento de los valores anteriores. Utilice Math.random() para las tiradas de dados del juego; utilice crypto.getRandomValues() para contraseñas, tokens y claves de cifrado. Diferencia de rendimiento: ~100 veces más rápido para Math.random() pero la diferencia de seguridad es infinita (predecible versus impredecible).
Los PRNG son deterministas: la misma semilla siempre produce la misma secuencia. Observar suficientes salidas (624 para Mersenne Twister) permite predecir todos los valores futuros. Los TRNG y CSPRNG generados a partir de la entropía del hardware son computacionalmente impredecibles: ningún algoritmo conocido puede predecir la siguiente salida a partir de valores anteriores sin conocer el estado interno. Sin embargo, las implementaciones defectuosas (el defectuoso OpenSSL de Debian en 2008, que redujo la entropía a 32.767 posibilidades) pueden volverse predecibles. Utilice siempre bibliotecas RNG bien auditadas.
Su RNG tiene un valor fijo (posiblemente por defecto 0 o 1). Muchos idiomas utilizan una semilla predeterminada para la reproducibilidad si no especifica una. En Python, llame a random.seed() sin argumentos para generar desde la hora del sistema. En C++, use std::random_device para inicializar. En JavaScript, Math.random() se genera automáticamente al cargar la página. Para obtener resultados reproducibles (pruebas, investigación), utilice intencionalmente una semilla fija como random.seed(42). Para obtener resultados impredecibles, asegúrese de sembrar desde fuentes de alta entropía.
