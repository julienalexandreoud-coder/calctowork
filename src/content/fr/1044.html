Qu'est-ce que le rapport de contraste ?

Le rapport de contraste mesure la différence de luminance entre les parties les plus claires et les plus sombres d'une image ou d'une interface. Un rapport de contraste de 21:1 représente un contraste maximum (blanc pur sur noir pur), tandis que 1:1 représente aucun contraste (couleurs identiques). Les directives pour l'accessibilité du contenu Web (WCAG) exigent des taux de contraste minimaux pour un texte lisible.

Les concepteurs Web utilisent des taux de contraste pour garantir que le texte est lisible pour les utilisateurs malvoyants. Le texte noir (#000000) sur fond blanc (#FFFFFF) atteint un contraste de 21:1, dépassant de loin l'exigence WCAG AA de 4,5:1 pour un texte normal. Le texte gris clair (#999999) sur blanc n'atteint que 2,8:1 – ce qui ne répond pas aux normes d'accessibilité et est difficile à lire pour de nombreux utilisateurs.

Les développeurs d'interface utilisateur vérifient les rapports de contraste pour les éléments interactifs tels que les boutons et les liens. Un bouton bleu (#4A90D9) avec du texte blanc atteint un contraste de 4,6:1 – dépassant les WCAG AA. Le même bleu avec du texte gris clair (#CCCCCC) tombe à 1,8 : 1 – échouant et potentiellement inutilisable pour les utilisateurs malvoyants ou daltoniens.

Formules de rapport de contraste avec calculs travaillés

WCAG 2.1 définit le rapport de contraste en utilisant la luminance relative :

CR = (L₁ + 0,05) / (L₂ + 0,05)

Où L₁ est la luminance relative de la couleur la plus claire (0 à 1) et L₂ est la luminance relative de la couleur la plus foncée (0 à 1).

La luminance relative est calculée à partir des valeurs sRVB :

L = 0,2126 × R + 0,7152 × G + 0,0722 × B

Où R, G et B sont les valeurs de couleur linéarisées (corrigées gamma).

Exemple concret :Calculez le contraste entre #3366CC (bleu) et #FFFFFF (blanc).

Tout d'abord, convertissez l'hexadécimal en sRGB (échelle 0-255) : #3366CC = rgb(51, 102, 204), #FFFFFF = rgb(255, 255, 255).

Normaliser sur une échelle de 0 à 1 : R = 51/255 = 0,2, G = 102/255 = 0,4, B = 204/255 = 0,8 pour le bleu. Le blanc est R = G = B = 1.

Linéariser chaque canal (correction gamma inverse) :

Si valeur ≤ 0,03928 : linéaire = valeur / 12,92

Si valeur > 0,03928 : linéaire = ((valeur + 0,055) / 1,055)^2,4

Pour le bleu : R = ((0,2 + 0,055) / 1,055)^2,4 = 0,0337, G = ((0,4 + 0,055) / 1,055)^2,4 = 0,1369, B = ((0,8 + 0,055) / 1,055)^2,4 = 0,6072

Pour le blanc : R = G = B = ((1 + 0,055) / 1,055)^2,4 = 1,0

Calculer la luminance relative :

L_bleu = 0,2126 × 0,0337 + 0,7152 × 0,1369 + 0,0722 × 0,6072 = 0,0072 + 0,0979 + 0,0438 = 0,1489

L_blanc = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 1 = 1,0

Calculer le rapport de contraste :

CR = (1,0 + 0,05) / (0,1489 + 0,05) = 1,05 / 0,1989 = 5,28:1

Résultat : bleu #3366CC sur blanc #FFFFFF atteint un contraste de 5,28 : 1 – dépasse les normes WCAG AA (4,5 : 1) et AAA (3 : 1) pour le texte normal.

Comment calculer le rapport de contraste : 6 étapes

  1. Convertir les couleurs en RVB :Votre couleur de premier plan est #7B2D8E (violet) et votre arrière-plan est #F5F5F5 (gris clair). Convertir : #7B2D8E = RVB (123, 45, 142), #F5F5F5 = RVB (245, 245, 245).
  2. Normaliser sur une échelle de 0 à 1 :Violet : R = 123/255 = 0,482, G = 45/255 = 0,176, B = 142/255 = 0,557. Gris : R = G = B = 245/255 = 0,961.
  3. Appliquer la correction gamma :Pour le violet : R = ((0,482 + 0,055) / 1,055)^2,4 = 0,203, G = ((0,176 + 0,055) / 1,055)^2,4 = 0,026, B = ((0,557 + 0,055) / 1,055)^2,4 = 0,272. Pour le gris : R = G = B = ((0,961 + 0,055) / 1,055)^2,4 = 0,919.
  4. Calculer la luminance relative :L_violet = 0,2126 × 0,203 + 0,7152 × 0,026 + 0,0722 × 0,272 = 0,043 + 0,019 + 0,020 = 0,082. L_gray = 0,2126 × 0,919 + 0,7152 × 0,919 + 0,0722 × 0,919 = 0,919.
  5. Identifiez le plus clair et le plus foncé :L_gray (0,919) > L_purple (0,082), donc L₁ = 0,919 et L₂ = 0,082.
  6. Appliquer la formule de contraste :CR = (0,919 + 0,05) / (0,082 + 0,05) = 0,969 / 0,132 = 7,34:1. Ce violet sur gris clair atteint 7,34:1 – passe les normes WCAG AAA (7:1) pour un texte normal.

5 Exemples de rapport de contraste

Exemple 1 — Noir sur blanc (contraste maximal) :#000000 sur #FFFFFF. L_black = 0 (tous les canaux se linéarisent à 0). L_blanc = 1. CR = (1 + 0,05) / (0 + 0,05) = 1,05 / 0,05 = 21:1. Il s’agit du contraste maximal possible dans l’espace colorimétrique sRGB – la référence en matière de lisibilité.

Exemple 2 — Gris sur gris (contraste défaillant) :#999999 sur #CCCCCC. L_lumière = 0,2126 × 0,502 + 0,7152 × 0,502 + 0,0722 × 0,502 = 0,502. L_dark = 0,2126 × 0,282 + 0,7152 × 0,282 + 0,0722 × 0,282 = 0,282. CR = (0,502 + 0,05) / (0,282 + 0,05) = 0,552 / 0,332 = 1,66:1. Cela échoue à tous les niveaux WCAG – ce qui est courant dans les interfaces mal conçues où le texte « subtil » gris sur gris devient illisible.

Exemple 3 — Vert sur blanc (limite AA) :#00AA00 (vert vif) sur #FFFFFF. L_vert = 0,2126 × 0 + 0,7152 × 0,331 + 0,0722 × 0 = 0,237. L_blanc = 1. CR = (1 + 0,05) / (0,237 + 0,05) = 1,05 / 0,287 = 3,66:1. Cela échoue aux WCAG AA (4,5 : 1) pour le texte normal, mais réussit pour le texte volumineux (18 pts ou 14 pts en gras, nécessitant 3 : 1). Le texte vert est particulièrement problématique pour les utilisateurs daltoniens rouge-vert.

Exemple 4 — Jaune sur noir (contraste élevé) :#FFFF00 (jaune pur) sur #000000. L_jaune = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 0 = 0,928. L_noir = 0. CR = (0,928 + 0,05) / (0 + 0,05) = 0,978 / 0,05 = 19,56:1. Le jaune sur le noir permet d'obtenir un contraste presque maximal. C'est pourquoi les panneaux d'avertissement et les étiquettes de danger utilisent cette combinaison pour une visibilité maximale.

Exemple 5 — Lien bleu sur blanc (modèle Web commun) :#0000EE (lien standard bleu) sur #FFFFFF. L_bleu = 0,2126 × 0 + 0,7152 × 0,033 + 0,0722 × 0,549 = 0,024 + 0,040 = 0,064. L_blanc = 1. CR = (1 + 0,05) / (0,064 + 0,05) = 1,05 / 0,114 = 9,21:1. Le lien bleu traditionnel passe facilement tous les niveaux WCAG – une des raisons pour lesquelles il est devenu la norme du Web dans les années 1990 et reste accessible aujourd'hui.

4 erreurs courantes liées au rapport de contraste

  • Utilisation de formules de luminance simples :Le calcul de la luminance comme (R + V + B) / 3 ignore la sensibilité différente de la vision humaine à chaque couleur. Le vert contribue à 71,52 % de la luminosité perçue, le rouge à 21,26 %, le bleu seulement à 7,22 %. Une couleur comme #00FF00 (vert pur) apparaît beaucoup plus lumineuse que #0000FF (bleu pur) malgré des sommes RVB égales.
  • Sauter la correction gamma :L’utilisation directe des valeurs sRVB sans linéarisation produit une luminance incorrecte. L'espace colorimétrique sRVB utilise une courbe gamma pour correspondre à la perception humaine. #808080 (gris moyen) a une luminance linéaire de 0,216 et non de 0,5 – une différence de 57 % qui affecte de manière significative les calculs de contraste.
  • Tester uniquement les tailles de texte normales :Les WCAG ont des exigences différentes pour le texte normal (4,5:1 AA, 7:1 AAA) et le texte volumineux (3:1 AA, 4,5:1 AAA). Le texte de grande taille est en gras de 18 ou 14 pts. Un contraste de 3,5 : 1 échoue à AA pour un corps de texte de 12 pts, mais réussit pour des titres de 20 pts.
  • Ignorer les éléments non textuels :WCAG nécessite également un contraste de 3:1 pour les composants de l'interface utilisateur (boutons, bordures de saisie) et les objets graphiques (icônes, graphiques). Une bordure de bouton grise (#CCCCCC) sur fond blanc (#FFFFFF) n'atteint que 1,3 : 1, ce qui échoue à l'exigence de 3 : 1 et rend le bouton invisible pour de nombreux utilisateurs.

5 conseils pour respecter le rapport de contraste

  • Commencez avec des valeurs par défaut à contraste élevé :Utilisez #000000 sur #FFFFFF (21:1) ou #1A1A1A sur #FAFAFA (18:1) comme base. Ceux-ci dépassent toutes les exigences et offrent une marge pour les variations de conception. Réduisez intentionnellement le contraste uniquement pour des éléments de conception spécifiques, jamais pour le corps du texte.
  • Utilisez les DevTools du navigateur :Chrome et Firefox DevTools affichent les taux de contraste dans le sélecteur de couleurs. Cliquez sur n’importe quelle couleur dans le panneau Styles et le sélecteur affiche le rapport de contraste par rapport à l’arrière-plan. Il montre également en un coup d'œil la réussite/l'échec pour les WCAG AA et AAA.
  • Test avec des simulateurs de daltonisme :Des outils tels que Stark, Color Oracle ou les paramètres de rendu de Chrome simulent divers types de déficiences de vision des couleurs. Un contraste rouge-vert qui passe mathématiquement (4,5 : 1) peut toujours être illisible pour les utilisateurs deutéranopes si la teinte est le seul facteur distinctif.
  • Vérifiez les états de survol et de focus :Un bouton avec un contraste de 5:1 dans son état par défaut peut chuter à 2:1 au survol si l'arrière-plan s'éclaircit. Testez tous les états interactifs : par défaut, survol, focus, actif et désactivé. Chaque État doit répondre aux exigences minimales de contraste.
  • Utilisez des outils d'audit automatisés :Lighthouse (intégré à Chrome), ax DevTools ou WAVE peuvent analyser des pages entières à la recherche de violations de contraste. Ces outils détectent les problèmes manqués lors des tests manuels, comme le contenu généré dynamiquement ou les variables CSS qui modifient le contraste en fonction du thème.

4 FAQ sur le rapport de contraste

WCAG 2.1 niveau AA nécessite 4,5 : 1 pour le texte normal, 3 : 1 pour le texte volumineux (18 pts ou 14 pts en gras) et 3 : 1 pour les composants et les graphiques de l'interface utilisateur. Le niveau AAA nécessite 7:1 pour le texte normal, 4,5:1 pour le texte volumineux et 4,5:1 pour les composants de l'interface utilisateur. La plupart des organisations ciblent les AA comme norme de conformité légale.

Pas directement. Les WCAG ne nécessitent pas de rapports de contraste spécifiques dans les photographies ou les illustrations. Cependant, le texte superposé aux images doit répondre aux exigences de contraste, soit en ajoutant une superposition semi-transparente derrière le texte, soit en utilisant des ombres de texte. Les images décoratives nécessitent un alt="" et les images informatives nécessitent un texte alternatif descriptif.

Tout d’abord, composez la couleur transparente sur son arrière-plan à l’aide de la fusion alpha. Pour rgba(0, 0, 0, 0.5) sur #FFFFFF : RVB résultant = (0 × 0,5 + 255 × 0,5) = 127,5 → #808080. Calculez ensuite le contraste entre la couleur composée (#808080) et l'arrière-plan final. La transparence elle-même n'affecte pas le contraste : seule la couleur finale visible compte.

La constante de 0,05 tient compte de l'éblouissement de l'écran et des réflexions de la lumière ambiante. Dans des conditions de visualisation réelles, même le « noir pur » sur un écran reflète un peu de lumière, et le « blanc pur » n’apparaît pas infiniment brillant. Le décalage empêche les rapports de contraste infinis (divisant par zéro) et modélise les environnements de visualisation typiques avec une lumière ambiante modérée.
