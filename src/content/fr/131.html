Qu'est-ce qu'un logarithme ?

Un logarithme répond à la question : « À quelle puissance dois-je élever un nombre de base pour obtenir un autre nombre ? Par exemple, log₂(8) = 3 car 2³ = 8. Les logarithmes transforment la multiplication en addition, rendant les calculs complexes gérables. Cette calculatrice calcule des logarithmes pour n'importe quelle base de 2 à 100, vous montrant exactement comment l'exposant se rapporte à vos valeurs d'entrée.

Les logarithmes ont été inventés par John Napier en 1614 pour simplifier les calculs astronomiques. Avant les ordinateurs, les scientifiques utilisaient des tables de logarithmes pour multiplier de grands nombres en additionnant leurs journaux. Aujourd’hui, les logarithmes restent incontournables en informatique (la recherche binaire est O(log n)), en acoustique (décibels), en chimie (échelle de pH) et en sismologie (échelle de Richter).

La formule du logarithme estlog_b(x) = y, ce qui signifieb^y = x. Si vous connaissez la base b et le résultat x, le logarithme vous donne l'exposant y. Par exemple, log₁₀(1000) = 3 car 10³ = 1000.

Formules expliquées

La calculatrice utilise la formule de changement de base pour calculer les logarithmes de n'importe quelle base :

log_b(x) = ln(x) / ln(b)

Où ln représente le logarithme népérien (base e ≈ 2,718). Cette formule fonctionne parce que les logarithmes de bases différentes sont proportionnels les uns aux autres.

Propriétés du logarithme clé :

  • Règle de produit : log_b(xy) = log_b(x) + log_b(y)
  • Règle de quotient : log_b(x/y) = log_b(x) - log_b(y)
  • Règle de puissance : log_b(x^n) = n × log_b(x)
  • log_b(1) = 0 pour toute base b
  • log_b(b) = 1 pour toute base b

Guide étape par étape

  1. Entrez le chiffre (x) :C'est la valeur dont vous souhaitez trouver le logarithme. Doit être positif (supérieur à 0). Exemple : 1000
  2. Entrez la base (b):C'est la base du logarithme. Les bases communes sont 2 (binaire), 10 (log commun) et e (log naturel). Exemple : 10
  3. Cliquez sur calculer :La calculatrice applique log_b(x) = ln(x) / ln(b)
  4. Lisez votre résultat :Pour log₁₀(1000), le résultat est exactement 3
  5. Verify:Vérifiez en calculant b^result = x. Pour notre exemple : 10³ = 1000 ✓

Exemples réels avec calculs

Exemple 1 : Logarithme binaire (informatique)
Rechercher log₂(256) : combien de bits sont nécessaires pour représenter 256 valeurs ?
log₂(256) = ln(256) / ln(2) = 5,545 / 0,693 = 8
Vérification : 2⁸ = 256 ✓
Application : Vous avez besoin de 8 bits (1 octet) pour stocker les valeurs 0 à 255.

Exemple 2 : Logarithme commun (notation scientifique)
Trouvez log₁₀(500) : Quelle puissance de 10 est égale à 500 ?
log₁₀(500) = ln(500) / ln(10) = 6,215 / 2,303 = 2,699
Vérification : 10^2,699 ≈ 500 ✓
Application : 500 = 10^2,699 = 5 × 10² en notation scientifique.

Exemple 3 : calcul du pH (chimie)
Si [H⁺] = 0,0001 mol/L, quel est le pH ?
pH = -log₁₀(0,0001) = -log₁₀(10⁻⁴) = -(-4) = 4
Application : pH 4 indique une solution acide (comme le jus de tomate).

Exemple 4 : Calcul des décibels (acoustique)
Rapport d'intensité sonore de 1000:1, quel est le niveau en dB ?
dB = 10 × log₁₀(1000) = 10 × 3 = 30 dB
Application : Une augmentation de 30 dB représente un changement d'intensité de 1 000 × (chuchoter à la bibliothèque).

Exemple 5 : Échelle de Richter (sismologie)
Le séisme A a une amplitude de 10 000 fois la référence. Ampleur?
M = log₁₀(10000) = 4
Application : tremblement de terre de magnitude 4,0 (légers tremblements, perceptibles à l'intérieur).

4 erreurs courantes

  • Essayer de calculer le journal de nombres nuls ou négatifs :Les logarithmes ne sont définis que pour des nombres réels positifs. log(0) n'est pas défini (s'approche de -∞) et log(-5) n'a pas de véritable solution.
  • Base de journal déroutante :log₂(8) = 3, mais log₁₀(8) = 0,903 et ln(8) = 2,079. Vérifiez toujours de quelle base vous avez besoin pour votre application.
  • Mauvaise application de la règle produit :log(x + y) ≠ log(x) + log(y). La règle du produit ne fonctionne que pour la multiplication : log(xy) = log(x) + log(y).
  • Oublier de diviser par ln(base):Lorsque vous utilisez des calculatrices qui n'ont que ln ou log₁₀, rappelez-vous log_b(x) = ln(x)/ln(b), pas seulement ln(x).

4 conseils de pro

  • Utilisez les propriétés du journal pour estimer :Si vous connaissez log₁₀(2) ≈ 0,301, alors log₁₀(200) = log₁₀(2×100) = 0,301 + 2 = 2,301. Le calcul mental devient plus facile.
  • Reconnaître les valeurs communes :Mémorisez log₁₀(2) ≈ 0,301, log₁₀(3) ≈ 0,477, log₂(10) ≈ 3,322. Ceux-ci apparaissent fréquemment dans l’ingénierie et l’informatique.
  • Vérifier le caractère raisonnable :Pour la base 10, log₁₀(1) = 0, log₁₀(10) = 1, log₁₀(100) = 2. Votre résultat doit se situer entre ces repères.
  • Utilisez des échelles logarithmiques pour de larges plages :Lorsque les données s'étendent sur des ordres de grandeur (comme les énergies sismiques de 10⁴ à 10¹⁸ joules), tracez sur une échelle logarithmique pour tout visualiser clairement.

FAQs

Le logarithme népérien utilise la base e ≈ 2,71828, le nombre d'Euler. Il apparaît naturellement dans les problèmes de croissance/décroissance, le calcul (la dérivée de ln(x) est 1/x) et la composition continue. ln(x) = log_e(x).
Pour les nombres réels, aucun exposant d’une base positive ne produit un résultat négatif. 2^y est toujours positif pour tout y réel. Cependant, il existe des logarithmes complexes de nombres négatifs utilisant des nombres imaginaires.
Utiliser log₂ en informatique (arbres binaires, structures de données, théorie de l'information en bits). Utilisez log₁₀ en science/ingénierie (notation scientifique, pH, décibels, échelle de Richter). Utilisez ln (base e) dans les modèles de calcul et de croissance continue.
Les logarithmes sont des fonctions inverses des exponentielles. Si y = b^x, alors x = log_b(y). Graphiquement, ils se reflètent sur la ligne y = x. Cette relation résout des équations telles que 2^x = 100 en prenant les logs des deux côtés.
