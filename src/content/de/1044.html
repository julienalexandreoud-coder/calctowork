Was ist das Kontrastverhältnis?

Das Kontrastverhältnis misst den Luminanzunterschied zwischen den hellsten und dunkelsten Teilen eines Bildes oder einer Schnittstelle. Ein Kontrastverhältnis von 21:1 bedeutet maximalen Kontrast (reines Weiß auf reinem Schwarz), während 1:1 keinen Kontrast (identische Farben) bedeutet. Die Web Content Accessibility Guidelines (WCAG) verlangen Mindestkontrastverhältnisse für lesbaren Text.

Webdesigner verwenden Kontrastverhältnisse, um sicherzustellen, dass Texte für Benutzer mit Sehbehinderungen lesbar sind. Schwarzer Text (#000000) auf weißem Hintergrund (#FFFFFF) erreicht einen Kontrast von 21:1 – weit über der WCAG AA-Anforderung von 4,5:1 für normalen Text. Hellgrauer Text (#999999) auf Weiß erreicht nur 2,8:1 – was die Barrierefreiheitsstandards verfehlt und für viele Benutzer schwer lesbar ist.

UI-Entwickler prüfen die Kontrastverhältnisse für interaktive Elemente wie Schaltflächen und Links. Eine blaue Schaltfläche (#4A90D9) mit weißem Text erreicht einen Kontrast von 4,6:1 und besteht damit die WCAG AA. Das gleiche Blau mit hellgrauem Text (#CCCCCC) sinkt auf 1,8:1 – ein Fehler und möglicherweise unbrauchbar für Benutzer mit Sehbehinderung oder Farbenblindheit.

Kontrastverhältnisformeln mit ausgearbeiteten Berechnungen

WCAG 2.1 definiert das Kontrastverhältnis anhand der relativen Luminanz:

CR = (L₁ + 0,05) / (L₂ + 0,05)

Dabei ist L₁ die relative Leuchtdichte der helleren Farbe (0 bis 1) und L₂ die relative Leuchtdichte der dunkleren Farbe (0 bis 1).

Die relative Leuchtdichte wird aus sRGB-Werten berechnet:

L = 0,2126 × R + 0,7152 × G + 0,0722 × B

Dabei sind R, G und B die linearisierten (gammakorrigierten) Farbwerte.

Arbeitsbeispiel:Berechnen Sie den Kontrast zwischen #3366CC (blau) und #FFFFFF (weiß).

Konvertieren Sie zunächst Hex in sRGB (Skala 0-255): #3366CC = rgb(51, 102, 204), #FFFFFF = rgb(255, 255, 255).

Auf eine Skala von 0-1 normalisieren: R = 51/255 = 0,2, G = 102/255 = 0,4, B = 204/255 = 0,8 für Blau. Weiß ist R = G = B = 1.

Linearisieren Sie jeden Kanal (inverse Gammakorrektur):

Wenn Wert ≤ 0,03928: linear = Wert / 12,92

Wenn Wert > 0,03928: linear = ((Wert + 0,055) / 1,055)^2,4

Für Blau: R = ((0,2 + 0,055) / 1,055)^2,4 = 0,0337, G = ((0,4 + 0,055) / 1,055)^2,4 = 0,1369, B = ((0,8 + 0,055) / 1,055)^2,4 = 0,6072

Für Weiß: R = G = B = ((1 + 0,055) / 1,055)^2,4 = 1,0

Berechnen Sie die relative Leuchtdichte:

L_blau = 0,2126 × 0,0337 + 0,7152 × 0,1369 + 0,0722 × 0,6072 = 0,0072 + 0,0979 + 0,0438 = 0,1489

L_white = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 1 = 1,0

Kontrastverhältnis berechnen:

CR = (1,0 + 0,05) / (0,1489 + 0,05) = 1,05 / 0,1989 = 5,28:1

Ergebnis: Blau #3366CC auf Weiß #FFFFFF erreicht einen Kontrast von 5,28:1 – übertrifft WCAG AA (4,5:1) und AAA (3:1) für normalen Text.

So berechnen Sie das Kontrastverhältnis: 6 Schritte

  1. Farben in RGB umwandeln:Ihre Vordergrundfarbe ist #7B2D8E (lila) und die Hintergrundfarbe ist #F5F5F5 (hellgrau). Convert: #7B2D8E = rgb(123, 45, 142), #F5F5F5 = rgb(245, 245, 245).
  2. Auf Maßstab 0-1 normalisieren:Lila: R = 123/255 = 0,482, G = 45/255 = 0,176, B = 142/255 = 0,557. Grau: R = G = B = 245/255 = 0,961.
  3. Gammakorrektur anwenden:Für Lila: R = ((0,482 + 0,055) / 1,055)^2,4 = 0,203, G = ((0,176 + 0,055) / 1,055)^2,4 = 0,026, B = ((0,557 + 0,055) / 1,055)^2,4 = 0,272. Für Grau: R = G = B = ((0,961 + 0,055) / 1,055)^2,4 = 0,919.
  4. Berechnen Sie die relative Leuchtdichte:L_lila = 0,2126 × 0,203 + 0,7152 × 0,026 + 0,0722 × 0,272 = 0,043 + 0,019 + 0,020 = 0,082. L_gray = 0,2126 × 0,919 + 0,7152 × 0,919 + 0,0722 × 0,919 = 0,919.
  5. Heller und dunkler identifizieren:L_gray (0,919) > L_purple (0,082), also L₁ = 0,919 und L₂ = 0,082.
  6. Kontrastformel anwenden:CR = (0,919 + 0,05) / (0,082 + 0,05) = 0,969 / 0,132 = 7,34:1. Dieses Lila auf Hellgrau erreicht 7,34:1 – übertrifft WCAG AAA (7:1) für normalen Text.

5 Beispiele für Kontrastverhältnisse

Beispiel 1 – Schwarz auf Weiß (maximaler Kontrast):#000000 auf #FFFFFF. L_black = 0 (alle Kanäle linearisieren auf 0). L_white = 1. CR = (1 + 0,05) / (0 + 0,05) = 1,05 / 0,05 = 21:1. Dies ist der maximal mögliche Kontrast im sRGB-Farbraum – der Goldstandard für Lesbarkeit.

Beispiel 2 – Grau in Grau (fehlender Kontrast):#999999 auf #CCCCCC. L_Licht = 0,2126 × 0,502 + 0,7152 × 0,502 + 0,0722 × 0,502 = 0,502. L_dunkel = 0,2126 × 0,282 + 0,7152 × 0,282 + 0,0722 × 0,282 = 0,282. CR = (0,502 + 0,05) / (0,282 + 0,05) = 0,552 / 0,332 = 1,66:1. Dies schlägt bei allen WCAG-Stufen fehl – ​​häufig bei schlecht gestalteten Benutzeroberflächen, bei denen „subtiler“ Grau-in-Grau-Text unleserlich wird.

Beispiel 3 – Grün auf Weiß (Grenzwert AA):#00AA00 (hellgrün) auf #FFFFFF. L_grün = 0,2126 × 0 + 0,7152 × 0,331 + 0,0722 × 0 = 0,237. L_white = 1. CR = (1 + 0,05) / (0,237 + 0,05) = 1,05 / 0,287 = 3,66:1. Dies besteht bei normalem Text nicht nach WCAG AA (4,5:1), reicht aber bei großem Text (18pt oder 14pt fett, erfordert 3:1) aus. Grüner Text ist besonders für rot-grün-blinde Benutzer problematisch.

Beispiel 4 – Gelb auf Schwarz (hoher Kontrast):#FFFF00 (reines Gelb) auf #000000. L_gelb = 0,2126 × 1 + 0,7152 × 1 + 0,0722 × 0 = 0,928. L_schwarz = 0. CR = (0,928 + 0,05) / (0 + 0,05) = 0,978 / 0,05 = 19,56:1. Gelb auf Schwarz erzielt einen nahezu maximalen Kontrast – weshalb Warnschilder und Gefahrenschilder diese Kombination für maximale Sichtbarkeit verwenden.

Beispiel 5 – Blauer Link auf Weiß (übliches Webmuster):#0000EE (Standardlink blau) auf #FFFFFF. L_blau = 0,2126 × 0 + 0,7152 × 0,033 + 0,0722 × 0,549 = 0,024 + 0,040 = 0,064. L_white = 1. CR = (1 + 0,05) / (0,064 + 0,05) = 1,05 / 0,114 = 9,21:1. Herkömmliches Link Blue besteht problemlos alle WCAG-Stufen – einer der Gründe, warum es in den 1990er Jahren zum Webstandard wurde und auch heute noch zugänglich ist.

4 häufige Fehler beim Kontrastverhältnis

  • Mit einfachen Leuchtdichteformeln:Bei der Berechnung der Leuchtdichte als (R + G + B) / 3 wird die unterschiedliche Empfindlichkeit des menschlichen Sehvermögens für jede Farbe ignoriert. Grün trägt 71,52 % zur wahrgenommenen Helligkeit bei, Rot 21,26 %, Blau nur 7,22 %. Eine Farbe wie #00FF00 (reines Grün) erscheint trotz gleicher RGB-Summen viel heller als #0000FF (reines Blau).
  • Gammakorrektur überspringen:Die direkte Verwendung von sRGB-Werten ohne Linearisierung führt zu einer falschen Luminanz. The sRGB color space uses a gamma curve to match human perception. #808080 (Mittelgrau) hat eine lineare Luminanz von 0,216, nicht 0,5 – ein Unterschied von 57 %, der sich erheblich auf die Kontrastberechnungen auswirkt.
  • Nur normale Textgrößen testen:WCAG stellt unterschiedliche Anforderungen an normalen Text (4,5:1 AA, 7:1 AAA) und großen Text (3:1 AA, 4,5:1 AAA). Großer Text ist 18pt oder 14pt fett. Bei einem Kontrast von 3,5:1 besteht die AA-Bewertung für 12-pt-Textkörper nicht, für 20-pt-Überschriften jedoch.
  • Nicht-Text-Elemente ignorieren:WCAG erfordert außerdem einen 3:1-Kontrast für UI-Komponenten (Schaltflächen, Eingaberahmen) und grafische Objekte (Symbole, Diagramme). Ein grauer Schaltflächenrand (#CCCCCC) auf weißem Hintergrund (#FFFFFF) erreicht nur 1,3:1 – was die 3:1-Anforderung nicht erfüllt und die Schaltfläche für viele Benutzer unsichtbar macht.

5 Tipps zur Einhaltung des Kontrastverhältnisses

  • Beginnen Sie mit den Standardeinstellungen für hohen Kontrast:Verwenden Sie als Basis #000000 auf #FFFFFF (21:1) oder #1A1A1A auf #FAFAFA (18:1). Diese übertreffen alle Anforderungen und bieten Spielraum für Gestaltungsvarianten. Reduzieren Sie den Kontrast nur absichtlich für bestimmte Designelemente, niemals für den Fließtext.
  • Verwenden Sie die DevTools des Browsers:Chrome und Firefox DevTools zeigen Kontrastverhältnisse im Farbwähler an. Klicken Sie im Bedienfeld „Stile“ auf eine beliebige Farbe, und die Auswahl zeigt das Kontrastverhältnis zum Hintergrund an. Außerdem wird auf einen Blick das Bestehen/Nichtbestehen der WCAG AA und AAA angezeigt.
  • Test mit Farbenblindheitssimulatoren:Tools wie Stark, Color Oracle oder die Rendering-Einstellungen von Chrome simulieren verschiedene Arten von Farbsehschwächen. Ein Rot-Grün-Kontrast, der mathematisch besteht (4,5:1), kann für deuteranopische Benutzer immer noch unlesbar sein, wenn der Farbton der einzige Unterscheidungsfaktor ist.
  • Schwebe- und Fokusstatus prüfen:Eine Schaltfläche mit einem Kontrast von 5:1 im Standardzustand kann beim Schweben auf 2:1 sinken, wenn der Hintergrund heller wird. Testen Sie alle interaktiven Zustände: Standard, Hover, Fokus, aktiv und deaktiviert. Jeder Staat muss Mindestkontrastanforderungen erfüllen.
  • Verwenden Sie automatisierte Audit-Tools:Lighthouse (in Chrome integriert), ax DevTools oder WAVE können ganze Seiten auf Kontrastverstöße scannen. Diese Tools erkennen Probleme, die bei manuellen Tests übersehen werden, etwa dynamisch generierte Inhalte oder CSS-Variablen, die den Kontrast je nach Thema ändern.

4 FAQs zum Kontrastverhältnis

WCAG 2.1 Level AA erfordert 4,5:1 für normalen Text, 3:1 für großen Text (18pt oder 14pt fett) und 3:1 für UI-Komponenten und Grafiken. Stufe AAA erfordert 7:1 für normalen Text, 4,5:1 für großen Text und 4,5:1 für UI-Komponenten. Die meisten Organisationen streben AA als Standard zur Einhaltung gesetzlicher Vorschriften an.

Nicht direkt. WCAG verlangt keine spezifischen Kontrastverhältnisse innerhalb von Fotos oder Illustrationen. Auf Bildern überlagerter Text muss jedoch den Kontrastanforderungen genügen – entweder durch Hinzufügen einer halbtransparenten Überlagerung hinter dem Text oder durch die Verwendung von Textschatten. Dekorative Bilder benötigen alt="" und informative Bilder benötigen beschreibenden Alternativtext.

Kombinieren Sie zunächst die transparente Farbe mithilfe der Alpha-Mischung über dem Hintergrund. Für rgba(0, 0, 0, 0,5) über #FFFFFF: resultierendes RGB = (0 × 0,5 + 255 × 0,5) = 127,5 → #808080. Berechnen Sie dann den Kontrast zwischen der zusammengesetzten Farbe (#808080) und dem endgültigen Hintergrund. Die Transparenz selbst hat keinen Einfluss auf den Kontrast – nur die endgültige sichtbare Farbe ist wichtig.

Die 0,05-Konstante berücksichtigt Bildschirmspiegelungen und Umgebungslichtreflexionen. Unter realen Betrachtungsbedingungen reflektiert selbst „reines Schwarz“ auf einem Bildschirm etwas Licht, und „reines Weiß“ erscheint nicht unendlich hell. Der Versatz verhindert unendliche Kontrastverhältnisse (Division durch Null) und modelliert typische Betrachtungsumgebungen mit mäßigem Umgebungslicht.
